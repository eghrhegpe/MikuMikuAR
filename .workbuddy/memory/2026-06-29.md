## Externalize Babylon.js + 性能调优

### 问题
前端构建时间 ~74s（主要瓶颈：Babylon.js 6.8MB + 3171 模块），因此需要获取

### 方案：Externalize @babylonjs/core + @babylonjs/materials

**生产构建**：
- `vite.config.ts` 中 `external` 排除 Babylon 包，`format: 'iife'` 用 `output.globals` 映射到 `BABYLON`（关键：ES 格式下 globals 不生效，必须 IIFE）
- `index.html` 中通过 `<script src="/lib/babylon.js">` 加载 CDN UMD（8.3MB）

**开发构建（wails dev）**：
- 通过 `transformIndexHtml` 插件自动移除 UMD script 标签（避免冗余加载 8.3MB）
- `@babylonjs/core` 保持 `optimizeDeps.exclude`（因 `.fx` shader 文件 ESBuild 不支持）
- Vite dev server 直接 serve node_modules 的 ESM 模块

### 性能问题：按钮点击卡顿 2s

根因：`setEnvState()` → `applyEnvState()` 每次都全量重建环境资源：
- `createClouds()`：`perlin2()` 524,288 次调用，每次重新创建 256 元素排列数组 → ~1.5s
- `createWater()`：每次销毁重建 WaterMaterial → ~200ms
- `createParticleEmitter()`：每次销毁重建 GPU Particle 系统 → ~100ms

**修复**：
1. ✅ `perlin2()` 排列数组提升为模块级常量 `_perlinPerm`，抽取辅助函数 → 524,288 次调用从 ~1.5s 降为 ~20ms
2. ✅ `createClouds()` 已存在则只更新位置/缩放，不重新生成 512x512 噪声纹理
3. ✅ `createWater()` 已存在则直接更新参数（颜色、透明度、波浪等），不重建
4. ✅ `createParticleEmitter()` 已存在则直接跳过重建

### 效果
- 构建时间：~74s → **~1s**（98.7% 提升）
- 模块数：3171 → 127（96% 减少）
- 按钮点击：~2s → **~20-100ms**（跳过重建后）

### 变更文件
- `frontend/vite.config.ts` — 全面重写，Externalize + IIFE + 插件
- `frontend/index.html` — 添加 Babylon UMD script 标签（head）
- `public/lib/babylon.js` — **新增** CDN 下载 8.3MB

## SlideMenu `visibility:hidden` → `height:0` 高度修正

### 问题
上一版用 `visibility:hidden` 替代 `display:none` 解决了高度塌陷（`display:none` 导致 slide-inner 塌陷），
但引入了新问题：`visibility:hidden` 的 panel 仍占据 flex 空间 → 菜单高度 = max(两个 panel 的高度)，
不会随层级切换自动适配。

### 修复
改用 `height:0; overflow:hidden` 策略：
- 旧 panel：立即 `height:0; overflow:hidden`，维持 flex 占位（不破坏 side-by-side 布局），但高度为 0
- 新 panel：`cssText=""` 清除旧样式，恢复自然高度
- `slide-inner` 高度 = max(0, 新 panel 高度) = 新 panel 高度 ✅
- 不需要 cleanup 时的 `translateX(0)` snap（inner transform 保持原位）

### 变更
- `menu.ts` `animateSlide()` — 旧 panel `height:0; overflow:hidden`；新 panel `opacity` 淡入（延迟 50% 动画时长）
- `menu.ts` `reset()` — panel1 初始化为 `height:0; overflow:hidden; opacity:0`

### 构建验证

## SlideMenu 全面重构：叠层淡入淡出

### 动因
`height:0` 方案虽能工作，但用户诊断出四大结构性问题：
1. 双面板侧滑（`translateX` / `width:200%`）→ 高度管理依赖 `height:0` 补丁，layout 脆弱
2. `setTimeout(ms+30)` 硬编码清理 → 浏览器掉帧时竞态
3. 键盘/鼠标焦点抢占同一视觉状态 → 鼠标移出后键盘高亮不恢复
4. 「物理滑动」动画与「页面切换」心理模型不匹配

### 重构内容

**CSS**（`app.css`）：
- `.slide-inner`：`display:flex; width:200%` → `display:grid`
- `.slide-panel`：`width:50%; flex-shrink:0` → `grid-area:1/1`（双面板叠在同一网格单元，高度自然取 max）
- 预设 `transition: opacity 150ms ease-out`，JS 只需触发 opacity 值变更

**JS**（`menu.ts` 完整重写）：
- 移除全部补丁：`translateX` / `inner.style.transform` / `height:0` / `overflow:hidden` / `visibility:hidden` / `display:none`
- 动画：纯 opacity 淡入淡出（old: 1→0, new: 0→1），无物理滑动
- 清理：`transitionend` 事件（`propertyName === "opacity"`）+ 50ms 安全超时兜底
- 面板状态：`setPanelActive()` 统一管理 opacity/zIndex/pointerEvents
- 高度回收：cleanup 后 `oldPanel.innerHTML = ""`，Grid 自动取新面板高度
- 键盘/鼠标焦点分离：`pendingKeyFocus` 暂存，`mouseleave` 恢复

## SlideMenu 最终重构：absolute 叠层 + translateX 滑动 + transitionend

### 动因
上一版 Grid 叠层 + 纯 opacity 淡入淡出仍有问题：Grid 取两面板 max 高度，动画期间高度锁定，淡入淡出在高度变化场景下体验不佳。用户给出完整替代方案。

### 核心改动

**JS**（`menu.ts` 完整重写为 ~370 行）：
- 布局：面板 `position:absolute; inset` 叠层，脱离文档流，高度不互相影响
- 高度管理：`push`/`pop` 期间 `viewport.style.height` 锁定 max(旧, 新)，动画结束后 `transition: height 0.15s` 平滑过渡到目标高度
- 动画：`translateX(±30px) + opacity` 方向性滑动（push 从右入，pop 从左入）
- 清理：`transitionend` 事件 + 200ms 安全超时兜底（极少数未触发时救命）
- `reset`/`reRender`：`viewport.style.height` 直接设为面板 `scrollHeight`
- `abortTransition()`：清除 timer，重置 `transitioning`

**CSS**（`app.css`）：
- `.slide-inner`：`display:grid; flex:1` → `position:relative; min-height:0`（死代码清理）
- `.slide-panel`：删除 `grid-area:1/1`，`will-change` 扩展为 `opacity, transform`

## SlideMenu 布局修复：display:block + offsetHeight + 最小高度兜底

### 问题
Absolute 面板脱离文档流后，`scrollHeight` 偶尔为 0（CSS 未完全布局），`viewport` 高度被设为 0，`overflow:hidden` 裁掉全部内容——菜单一片空白。

### 修复（用户提供完整修正版）
1. **`display: block` 内联强制**：`initPanelStyle` 中写死 `p.style.display = "block"`，`reset`/`push`/`pop`/`popTo` 中每次重申，防止全局 CSS 覆盖为 `none`
2. **`offsetHeight` 替代 `scrollHeight`**：强制回流后 `offsetHeight` 更可靠
3. **`Math.max(..., 100)` 高度兜底**：保证 viewport 至少 100px，避免 0 高度截断
4. **每次设高度前强制回流**：`void el.offsetHeight` 确保布局计算完成
