## Externalize Babylon.js + 性能调优

### 问题
前端构建时间 ~74s（主要瓶颈：Babylon.js 6.8MB + 3171 模块），因此需要获取

### 方案：Externalize @babylonjs/core + @babylonjs/materials

**生产构建**：
- `vite.config.ts` 中 `external` 排除 Babylon 包，`format: 'iife'` 用 `output.globals` 映射到 `BABYLON`（关键：ES 格式下 globals 不生效，必须 IIFE）
- `index.html` 中通过 `<script src="/lib/babylon.js">` 加载 CDN UMD（8.3MB）

**开发构建（wails dev）**：
- 通过 `transformIndexHtml` 插件自动移除 UMD script 标签（避免冗余加载 8.3MB）
- `@babylonjs/core` 保持 `optimizeDeps.exclude`（因 `.fx` shader 文件 ESBuild 不支持）
- Vite dev server 直接 serve node_modules 的 ESM 模块

### 性能问题：按钮点击卡顿 2s

根因：`setEnvState()` → `applyEnvState()` 每次都全量重建环境资源：
- `createClouds()`：`perlin2()` 524,288 次调用，每次重新创建 256 元素排列数组 → ~1.5s
- `createWater()`：每次销毁重建 WaterMaterial → ~200ms
- `createParticleEmitter()`：每次销毁重建 GPU Particle 系统 → ~100ms

**修复**：
1. ✅ `perlin2()` 排列数组提升为模块级常量 `_perlinPerm`，抽取辅助函数 → 524,288 次调用从 ~1.5s 降为 ~20ms
2. ✅ `createClouds()` 已存在则只更新位置/缩放，不重新生成 512x512 噪声纹理
3. ✅ `createWater()` 已存在则直接更新参数（颜色、透明度、波浪等），不重建
4. ✅ `createParticleEmitter()` 已存在则直接跳过重建

### 效果
- 构建时间：~74s → **~1s**（98.7% 提升）
- 模块数：3171 → 127（96% 减少）
- 按钮点击：~2s → **~20-100ms**（跳过重建后）

- `frontend/index.html` — 添加 Babylon UMD script 标签（head）
- `public/lib/babylon.js` — **新增** CDN 下载 8.3MB
