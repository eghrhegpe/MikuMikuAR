## 一、AI Coding（3 条）

### 1. Kimi K3 登顶 Frontend Code Arena，2.8T 参数开源权重引发路线之争
- **事件**：月之暗面于 7/17 发布 2.8 万亿参数模型 K3（基于 KDA 混合线性注意力 + AttnRes 注意力残差，896 专家激活 16 个），以 1679 分登顶 **Frontend Code Arena**，系首个在该前端编程榜超越一众闭源模型的开源/开放权重模型；马斯克两度公开点赞。发布 48 小时后因请求过载暂停 C 端新订阅，被类比为「DeepSeek 2.0 时刻」。7/21 官方否认为蒸馏模型，强调底层原创架构；同时陷入「开放权重削弱前沿回报」「高定价可持续性」争议，估值传闻超 300 亿美元。
- **为何关注**：前端生成式编程是 AI Coding 最贴近产品端的硬指标。K3 用开放权重正面挑战闭源，叠加「性能跃升 + 集群承压 + 路线辩论」，是观察国产模型能力水位与开源/闭源博弈的关键样本，对联邦内「前端生成辅助 / 设计稿转代码」方向有直接参照价值。

### 2. Google 同日发布三款 Gemini 新模型，含面向代码安全的 Flash Cyber
- **事件**：7/22 Google 发布 **Gemini 3.6 Flash**（强化代码/多模态/知识，输出价降约 17%）、**Gemini 3.5 Flash-Lite**（低成本高吞吐）、以及 **Gemini 3.5 Flash Cyber**——专门面向网络安全优化，结合 CodeMender 代码安全智能体，用于软件漏洞检测与修复。同时透露已启动 Gemini 4 预训练，旗舰 3.5 Pro 仍在内测。
- **为何关注**：把「编程能力」与「代码安全」打包进同一模型家族，是 Coding Agent 走向生产可用的必然拼图。对联邦的启示：工具链若引入 AI 辅助代码审计，安全智能体可作为 CI 前置关卡，而非事后人工 review。

### 3. Mistral Leanstral 1.5：对 AI 生成代码做形式化验证，可接入 CI/CD
- **事件**：Mistral AI 发布 Leanstral 1.5，定位「AI 生成代码的正确性安全层」。在 57 个开源仓库上标记 47 处违规属性与 11 个真实 Bug（其中 5 个此前 GitHub 未报告，含一处 varint 解码库整数溢出——debug 崩溃、release 静默损坏）。验证成本已低到可嵌入 CI/CD 流水线。
- **为何关注**：当 Coding Agent 开始「自主提交」，正确性保障从「人工 review + 测试」前移到「形式化验证」。这是把不可信的 Agent 产出变成可部署资产的关键基础设施，与联邦 AGENTS.md 中「信任本机改动、靠 PR review 兜底」的工程纪律高度互补——自动验证可显著减轻 review 负担。

## 二、具身智能（2 条）

### 4. 阿里达摩院 RynnBrain 1.1：122B 开源具身基座模型，跨构型统一动作空间
- **事件**：阿里达摩院将开源具身基座模型推至 **122B-A10B** 规模（当前公开最大开放权重档，HuggingFace 趋势 30↑）。相对 1.0 新增接触点预测，2B/9B 变体获原生 3D grounding；配套 **RynnBrain-VLA** 采用统一跨构型动作空间 + 构型专属 mask，已部署于 Unitree G1、Astribot-S1、天机无极三平台。在 VSI-Bench、MMSI、RefSpatial-Bench 三项空间推理基准超所有闭源/开源被测模型，真机策略初始化也优于 Qwen 系与主流通用 VLA。
- **为何关注**：具身智能的「预训练大模型 + 一脑多形」路线正在被头部厂商用开源权重坐实。对联邦「一脑多态 / 跨构型部署」架构有直接的范式参考——统一动作空间 + 构型 mask 正是我们规避「每换本体重写策略」的可复用思路。

### 5. NVIDIA × HuggingFace 共建开源机器人 AI 栈：GR00T 1.7 + LeRobot + 10T Agent Tokens + Cosmos 3 Edge
- **事件**：NVIDIA 与 HuggingFace 联合发布开源机器人基础模型计划：将 **Isaac GR00T 1.7**（推理型 VLA）与 Isaac Teleop 数据采集框架直接接入 HuggingFace **LeRobot** 生态，并预告 Cosmos 3 集成；同步推出 **Open Data for Agents**——超 10 万亿预训练 token、数百万后训练样本（含区域化合成 persona 与 Nemotron Post-Training v3 Prompt Atlas）。另发布 **Cosmos 3 Edge**：4B 参数世界模型，可在机器人端侧直接生成动作。目标是降低机器人 AI 训练/部署门槛，避免各家重复造轮子。
- **为何关注**：这是具身智能「基础设施标准化」的标志性动作——把数据、模型、仿真、端侧世界模型统一到被广泛使用的开源栈里。对联邦意味着：本体与策略研发可站在共享基座上，把精力集中在「场景闭环 / 数据飞轮 / 岗位化落地」而非重建底层；同时端侧世界模型（Cosmos 3 Edge）指向低延迟、可离线部署的本地推理，契合我们「本地优先 + 隐私」的工程取向。
