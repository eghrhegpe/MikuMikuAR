## 2. Agent Data Injection（ADI）：AI 编程安全范式的新威胁类别
**方向**：AI coding / 安全　**时间**：arXiv 2026-07-06 提交，本周密集讨论

- **事件**：研究者正式化 **Agent Data Injection Attacks** 一类间接提示注入——攻击者伪造机器解析的元数据或工具返回（资源标识符、JSON 字段、响应模板），让 Agent 把攻击者内容当作"可信数据"。论文在 Claude Code、OpenAI Codex、Gemini CLI 上验证成功，并指出：**针对自然语言指令注入的防御对 ADI 无效**，因为它滥用的是机器解析信道，而非自然语言信道。
- **值得关注**：
  - 防御重心从"指令注入"扩展到"数据来源可信性（data provenance）"——schema 校验、信道隔离、可信/不可信输入分离成为必需。
  - 对联邦的直接警示：任何把 AI Agent 接入构建/部署/CI 流水线的设计，必须把**结构化 Agent 输入（工具返回、JSON、资源 ID）一律视为不可信**，并保留 subagent 轨迹用于审计。

## 3. AI 编程进入"代理阶段"：横评显示 SWE-bench 准确率破 80%
**方向**：AI coding / 行业范式　**时间**：2026-07-20

- **事件**：覆盖 Claude Code v2.1、Cursor 3.0、Trae SOLO、Copilot、Windsurf 的五款工具深度横评发布，指出 AI 编程已从"补全"进化到"代理"阶段，**SWE-bench 准确率突破 80%**。同时观点文章厘清核心指标正从"代码生成准确率"转向"任务交付成功率"；Claude Code 桌面版 7 月已支持 **Windows 原生运行**，Trae SOLO 自主模式可 40 分钟完成 Swagger 文档全流程。
- **值得关注**：
  - 范式确认："Coding Agent 本质是超级程序员，交付的是能跑起来的结果"。这意味着联邦开发流程中，更多"构建→测试→修复"回路可放心交给 Agent。
  - 边界清醒：复杂长程任务稳定性仍是天花板，无人值守需配合自我验证链路（见历史 ADR / 06-20 的 OpenSquilla 0.4.0 自我验证）。
