- 构建脚本以 `set -euo pipefail` 或 `$ErrorActionPreference = "Stop"` 开头，遇到任意失败立即退出，保证流水线幂等。
- 所有脚本通过相对路径推导 `REPO_ROOT`（`$(cd "$(dirname ...)" && pwd)/..` 或 `Split-Path -Parent $PSScriptRoot`），再基于此拼接 go.mod、frontend、bin 等绝对路径，避免对调用工作目录的假设。
- 版本号来源单一：从 `package.json` 的 version 字段读取，并通过 sed 同步写入 `build/config.yml`，再由 `-ldflags -X main.AppVersion=...` 注入 Go 二进制。
- Go 构建统一启用 `mpr` build tag 以开启多线程物理（MPR），并在 production 模式下追加 `-s -w` 裁剪符号表。
- 契约测试脚本以 Python 标准库 pathlib + re 实现，返回非零退出码表示失败，供 CI 直接消费，不依赖第三方测试框架。
