本模块为纯脚本集合，按用途分为三类：
- 构建入口：`scripts/wails/build.ps1` 作为 Windows 主入口，顺序执行「前置依赖检查 → 契约测试 → Go 测试 → Go 编译 → 前端 npm build → 产物校验」五步流水线；各平台独立脚本 `build-{linux,darwin,windows,android,ios}.sh(.ps1)` 分别封装对应平台的 go build tag（debug/mpr/production）、LDFLAGS 注入版本、Vite 前端构建与产物重命名逻辑。
- 资源生成器：`gen-textures.py` / `gen-textures-stdlib.py` 用 Pillow 程序化生成地面纹理贴图；`gen-icon-bundle.mjs` 扫描 frontend/src 中 lucide/tabler 图标引用并拉取 api.iconify.design 生成离线 bundle `icons-bundle.ts`；`gen_appicon.py` 生成应用图标。
- 契约测试：`tests/test_config_syntax.py` 校验 wails.json/Taskfile.yml 与 go.mod 语法；`tests/test_markdown_links.py` 扫描 docs 内 Markdown 链接有效性；`tests/ai_mistake_tracker.py` 追踪 AI 生成错误。
依赖方向单向向外：脚本读取仓库根目录的 go.mod、frontend/package.json、docs 等源码文件，输出到 bin/dist/frontend/public 等产物目录，不引入任何运行时依赖。
