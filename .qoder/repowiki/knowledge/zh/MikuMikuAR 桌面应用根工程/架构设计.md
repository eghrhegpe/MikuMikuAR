根目录作为 Wails v3 应用的唯一入口：`main.go` 通过 `//go:embed all:frontend/dist` 将前端产物内嵌为只读资产，并注册 `app.NewApp` 提供的 Service；`application.AssetOptions.Handler` 注入 COOP/COEP 中间件，使 WebView2 文档具备 SharedArrayBuffer 能力，从而驱动 Babylon.js 中的多线程 MMD WASM。

- 运行时耦合：Go 后端（`go_backend`）通过 `internal/app` 暴露给 Wails 的 Service 接口，前端（`frontend_app`）经 Vite 构建后由同一进程托管，两者共享 `mikumikuar` Go module 命名空间。
- 构建编排：顶层 `Taskfile.yml` 仅声明 `build/package/run/dev/server/docker` 等门面任务，实际平台差异通过 `includes:` 委派到 `build/{windows,darwin,linux,ios,android}/Taskfile.yml`，由 `scripts_build` 子模块提供通用逻辑。
- 版本注入：`AppVersion` / `BuildTime` / `CommitHash` 三个变量统一通过 `-ldflags` 注入，供前后端在 UI 中展示诊断信息。
- E2E 调试：`start-e2e.ps1` 设置 `MMCAR_DEBUG_PORT` 环境变量，`main.go` 读取该变量向 WebView2 注入 `--remote-debugging-port`，使 Playwright 可 attach 到浏览器实例。
