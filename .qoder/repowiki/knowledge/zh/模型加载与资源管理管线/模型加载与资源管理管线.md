## 加载管线总览

MikuMikuAR 的模型加载管线以 `model-loader.ts`（688 行）为核心，完成从文件选择到场景渲染的全流程：

```
用户选择 PMX → loadPMXFile()
  → 竞态取消（AbortSignal.any）
  → 纹理收集（FileReader → base64）
  → PMX 解析（babylon-mmd）
  → Actor/Stage 实例化
  → 后处理（材质/物理/相机）
```

## 竞态取消机制（ADR-096）

```typescript
// frontend/src/scene/manager/model-loader.ts
const abortCtrl = new AbortController();
const signal = AbortSignal.any([externalSignal, abortCtrl.signal]);
```

- **外部信号**：用户切换模型、关闭面板
- **内部信号**：加载超时、解析失败
- 合并信号传入所有异步步骤，任意阶段可中断整条管线

## 资源类型与加载策略

| 资源类型 | 格式 | 加载方式 | 缓存策略 |
|----------|------|----------|----------|
| 3D 模型 | PMX/PMd | babylon-mmd 解析 | 内存缓存（LRU） |
| 动作数据 | VMD | 二进制解析 + Shift-JIS 解码 | 按需加载 |
| 纹理 | PNG/JPG/TGA | FileReader → base64 | 纹理缓存 |
| 音频 | WAV/MP3 | WebAudio decodeAudioData | 预加载 |
| 换装 | ZIP | Go 端解压 → HTTP 服务 | 惰性 cache |

## 材质系统

材质编辑采用 **节点图（Node Material）** 模式：

- `material-editor.ts` — 可视化编辑界面
- `material-presets.ts` — 预设材质库
- 支持 PBR / Toon / 自定义 Shader

## 换装系统（Outfit System, ADR-020）

```
ZIP 压缩包 → Go 端解压到 cache 目录
  → HTTP 服务暴露文件（隔离根 + 路径穿越防护）
  → 前端通过 URL 加载纹理/网格
  → 运行时替换 Actor 网格
```

关键约束：
- 换装 ZIP 内必须包含 `manifest.json`
- 纹理路径相对于 ZIP 根目录
- 支持多 outfit 叠加（分层渲染）

## 资源释放规则

| 场景 | 释放方式 |
|------|----------|
| 切换模型 | `disposeCurrentModel()` → 级联释放 Mesh/Texture/Material |
| 关闭应用 | `disposeRenderer()` → 释放全部 Babylon 资源 |
| 加载失败 | `catch` 块中释放已创建的中间资源 |
| 换装卸载 | 释放替换的 Mesh，保留原始 Actor |

## 开发者规则

| 规则 | 说明 |
|------|------|
| 新增加载步骤 | 必须在 `AbortSignal` 链中检查 `signal.aborted` |
| 资源创建 | 每个 `new Xxx()` 必须有对应 `dispose()` 路径 |
| 纹理加载 | 使用 `TextureLoader` 统一入口，不要直接 `new Texture()` |
| 错误处理 | 加载失败后 `finally` 中清理中间状态 |
