## 三层架构

环境系统采用 **Facade + Dispatcher + 子系统** 三层架构，解决循环依赖（ADR-138）并对外暴露单一入口：

```
外部模块
    ↓ import from env.ts
env.ts（Facade）
    ↓ 调用各子系统 apply 函数
env-dispatcher.ts（事件分发）
    ↓ registerEnvCallback() 注册的回调
各子系统（sky/ground/water/fog/clouds/particles/lighting）
```

## Dispatcher 模式（ADR-138）

```typescript
// frontend/src/scene/env/env-dispatcher.ts
const _callbacks: Array<(changed: Set<string>, state: EnvState) => void> = [];

export function registerEnvCallback(fn: EnvCallback) {
    _callbacks.push(fn);
}

export function dispatchEnvChange(changed: Set<string>, state: EnvState) {
    for (const cb of _callbacks) cb(changed, state);
}

// 每帧 tick 回调（用于动画/粒子等持续更新）
export function registerSceneTickCallback(cb: () => void) { ... }
```

**解决的问题**：`env-bridge.ts`（持久化层）不再需要 import 各子系统，打破循环依赖。

## Facade 模式

```typescript
// frontend/src/scene/env/env.ts — 对外唯一入口
export function applyEnvState(state: EnvState) {
    applySky(state);
    applyGround(state);
    applyFog(state);
    applyWater(state);
    applyParticles(state);
    applyClouds(state);
}
```

外部模块只 import `env.ts`，不直接调用子系统。

## 子系统清单

| 子系统 | 文件 | 职责 |
|--------|------|------|
| 天空盒 | `env-sky.ts` | HDR 环境贴图 + 旋转 + 曝光 |
| 地面 | `env-ground.ts` | 网格平面 + 纹理 + 反射 |
| 水面 | `env-water.ts` | Babylon WaterMaterial + 波纹参数 |
| 雾效 | `env-impl.ts` | 线性/指数雾 + 密度/颜色 |
| 云层 | `env-clouds.ts` | 粒子系统 / Billboard 云 |
| 粒子 | `env-particles.ts` | 雨/雪/樱花等环境粒子 |
| 光照 | `env-lighting.ts` | Directional + Hemispheric + Shadow |

## 状态驱动渲染

```typescript
// env-bridge.ts — 状态变化 → 分发 → 子系统响应
export function setEnvState(partial: Partial<EnvState>) {
    Object.assign(envState, partial);
    const changed = computeChangedKeys(partial);
    dispatchEnvChange(changed, envState);  // 通知子系统
    schedulePersistEnvState();              // 防抖持久化
}
```

子系统通过 `registerEnvCallback` 监听变化，只处理自己关心的 key。

## 环境上下文（env-context.ts）

```typescript
// 聚合当前所有环境子系统的运行时引用
export interface EnvContext {
    skyMaterial?: SkyMaterial;
    waterMaterial?: WaterMaterial;
    groundMesh?: Mesh;
    directionalLight?: DirectionalLight;
    // ...
}
```

用于 dispose 时级联释放、以及跨子系统的运行时查询。

## 开发者规则

| 规则 | 说明 |
|------|------|
| 新增环境参数 | 在 `env-state-schema.ts` 添加字段 + 默认值 |
| 新增子系统 | 实现 `apply(state)` + `registerEnvCallback(cb)` |
| 外部调用 | 只 import `env.ts`，不直接 import 子系统 |
| 资源释放 | `disposeEnvSystem()` 必须级联释放所有子系统的 RT/Material/Observer |
