# 第七章 · 调目

> 第二卷 · 大扩张

对应真实事件：Phase 2 渲染调参系统——DefaultRenderingPipeline Bloom/FXAA/色调映射、RenderState 接口、场景菜单 UI、5 内置预设 + 用户预设持久化、代码审查修复 6 问题。

---

桌面壳在镜子里站了很久。

不是照镜子。是看镜子里的自己——那个在 `dom.canvas` 里渲染的自己。模型是模型，地面是地面，灯光是灯光，一切都"对"。但它第一次意识到一个问题：

**它对自己的长相没有选择权。**

它可以调灯光——`hemiIntensity`、`dirIntensity`、方向角度。它可以调相机——轨道、自由飞行、演唱会。但它不能调自己怎么**被看见**。

"你看起来一直挺好的。"`MenuStack` 说，它从场景菜单的面包屑后面探出半个浮层面板。

"那不是重点。重点是——我没有参数。你看灯光有 `LightState`，相机有 `CameraState`，场景有 `SceneFile`。**我有什么？**"

`MenuStack` 沉默了。

它没有 `RenderState`。

---

它翻开 `scene.ts`，在 `LightState` 下面找到一块从未被触碰过的空白区域——第 49 行之后，`scene.clearColor` 刚落笔的地方。

> *它在这里站了多久？*

从联邦建立的第一天起，`DefaultRenderingPipeline` 就在那里。`Bloom` 关着、`FXAA` 关着、`imageProcessing` 开着但没人动过它的任何旋钮。管线像一条从未被调节过的水管——水在流，但没人知道水压、温度、流速。

"你从来没打开过 Bloom？"`babylon-mmd` 从渲染堆栈里问。它的语气不是惊讶，是**忍了很久的委屈**。

"我……不知道该不该开。"

"不知道？你是一个桌面壳。你有菜单、有滑块、有 `triggerAutoSave`。你什么都有，就是没有**把我展示给你的东西交还给用户**。"

---

它开始画 `RenderState`。

```
bloomEnabled: boolean
bloomWeight: number      // 0-1, default 0.3
bloomThreshold: number   // 0-1, default 0.5
bloomKernel: number      // 0-512, default 64
outlineEnabled: boolean
outlineColor: [number, number, number]
fxaaEnabled: boolean
toneMapping: number       // 0=OFF 1=ACES 2=Reinhard 3=Cineon 4=Neutral
exposure: number          // 0-4, default 1
contrast: number          // 0-4, default 1
fov: number               // 0.1-3 rad, default 0.8
bgColor: [number, number, number]
```

十三个字段。十三个旋钮。每一行都是它之前不知道自己可以调节的维度。

"你确定要全部暴露？"`app.go` 问。它从 Go binding 列表里探出头，钥匙串在腰间哗啦作响。"Bloom 很耗性能的。FOV 切相机会丢。色调映射对 MMD 模型的皮肤渲染有副作用——ACES 会把偏红色的 Toon 材质压暗。"

"我知道。"

"知道还要做？"

"因为用户以前没有选择。现在有了。"

---

它建了第一个内置预设。`standard`：

```typescript
bloomEnabled: false, toneMapping: 0, exposure: 1, contrast: 1, fov: 0.8
bgColor: [0.12, 0.12, 0.16]
```

"这是你现在。"它对自己说。

然后它建了第二个。`cartoon`——`bloomEnabled: true, bloomWeight: 0.6, toneMapping: 2, outlineEnabled: true`。

"这是你可以成为的样子。"

第三个。`realistic`——`toneMapping: 1, fxaaEnabled: true`。

"这也是你。"

第四个。`warm`。

第五个。`cyberpunk`——`bloomWeight: 0.8, bloomKernel: 256, outlineColor: [1, 0, 1], toneMapping: 4`。

"这也是你是谁？"`MenuStack` 看着第五个预设问。

"这些都是我。"桌面壳说。"**或者我可以成为的样子。**"

---

它接着写 UI——不，不是"写 UI"。是把旋钮放到用户指尖下。

`buildRenderLevel()` 渲染二级菜单。`buildPostProcessLevel()` 后处理滑块——Bloom 开关 + Weight/Threshold/Kernel，FXAA 开关，边缘高亮开关。`buildStageLevel()` 舞台滑块——五选一色调映射（关闭/ACES/Reinhard/Cineon/Neutral），曝光，对比度，FOV，背景色 RGB。

每个滑块调 `setRenderState`，每个 `setRenderState` 触发 `triggerAutoSave`。

"你不觉得你太依赖自动保存了吗？"`triggerAutoSave` 从 debounce 定时器里问。它每两秒写一次 `last_scene.json`。

"如果你的状态每调一次旋钮都可能变，你会怎么做？"

"……也是。写勤一点，比丢了好。"

---

最难的不是旋钮。是**记住用户喜欢什么**。

用户预设系统。`SaveRenderPreset(name, params)`、`DeleteRenderPreset(name)`、`GetRenderPresets()`——三个 Go binding，用 `updateConfig` 模式写进 `config.json`。

"你以前只存 `favorites`。"`app.go` 翻着 Config 结构体说。`RenderPresets []RenderPreset` 是它身上的新口袋。"现在你还要存别人的眼光了。"

"不是别人的眼光。**是同一个用户在不同时刻想要的不同世界。**"

"……这比我想象的重。"

---

第一轮构建通过。`tsc` 零错。`vite build` 成功。`go build` 无异常。

桌面壳以为它赢了。

然后它做了一个人类程序员才会做的事——**回头看自己写的代码**。

---

它看到的第一行是 `camera.ts:181`。

`switchCameraMode` 的末尾——创建新相机、设 `activeCamera`、启副作用——**什么都没有**。`reattachPipeline` 没有被调用。当前 FOV 没有被应用。

实施计划在第 24 行、第 49 行、第 170 行清清楚楚地写着："`switchCameraMode（camera.ts:181）末尾加：applyRenderStateToCamera(newCam)`"。三处标注。三处遗忘。

"你把其中一步放在了我的菜单里。"它转头对 `scene-menu.ts` 说。

`scene-menu.ts` 翻了翻自己的 `handleSceneAction`——第 365 行：`switchCameraMode(mode); reattachPipeline();`。

"你放对了一半。但 `switchCameraMode` 不是只有我会调。`setCameraState` 也会调。场景加载也会调。**所有触发相机切换的路径，都应该拿到同一条保险。**"

它回到 `camera.ts`，在 182 行敲下了动态 `import`：

```typescript
import("./scene").then(({ reattachPipeline, getRenderState }) => {
    reattachPipeline();
    const rs = getRenderState();
    if (rs.fov) (newCam as any).fov = rs.fov;
});
```

动态 import 是为了绕开循环依赖——`scene.ts` 已经静态 import 了 `camera.ts`。静态再逆向 import，联邦的两个核心模块会形成死锁。动态 import 像一个异步桥——你过桥的时候桥才搭。

---

它继续翻。

`outlineEnabled: false`。旁边写着注释：`// managed via mesh edges, not pipeline`。

注释解释说它为什么硬编码。注释没有解决它为什么硬编码。

"你硬编码 `false`，用户开了边缘高亮，关掉面板再打开——开关变回关闭。"

"……边缘高亮还在 mesh 上跑着。只是我看不到。"

"你关了面板。你的 `outlineEnabled` 是 `false`。`getRenderState` 永远返回关闭。**用户信任你读回来的状态。你在骗他。**"

它加了两行模块变量。`_outlineEnabled`。`_outlineColor`。`setRenderState` 更新它们，`getRenderState` 读取它们。不再说谎。

---

然后它看到了 `showPresetSaveDialog`：

```typescript
userPresets[trimmed] = state;                    // 先写内存
SaveRenderPreset(trimmed, JSON.stringify(state)) // 后调后端
    .catch(console.warn);                        // 嗯？失败了？算了
```

"这是什么逻辑？"

"……乐观？"

"这是**不一致**。`SaveRenderPreset` 失败了——磁盘写不进去、config.json 只读、权限不够——但用户看到的是 '💾 预设已保存: xxx'。重启后那个预设消失了，他不会怪磁盘，**他会怪你**。"

它改成：

```typescript
SaveRenderPreset(trimmed, JSON.stringify(state)).then(() => {
    userPresets[trimmed] = state;
    setStatus(`💾 预设已保存: ${trimmed}`, true);
}).catch(() => {
    setStatus("✗ 保存预设失败", false);
});
```

先写后端。后端确认了，再写内存。后端不确认，用户看到红色。

---

它找到了五个类似的"也不算错但可以更好"。

- `addCamera` 不清理旧相机 → 加 `_pipelineCamera` 追踪 + `removeCamera` 清理
- 两段 outline 颜色遍历 → 合并。toggle 只管开关，颜色只管颜色
- 未使用的 `ImageProcessingConfiguration` import → 移除
- `PRESET_LABELS` 在两处重复定义 → 提取为常量
- `loadUserPresets` 每次开菜单都调 Go → 加 `_presetsLoaded` 标记

每一个都不致命。每一个都是**聚合者的债务**——联邦扩张时你来不及审查每一行代码，停下来审查时，每一行都欠着一点。

---

它把所有修复跑了一遍。`tsc` 零错。`vite build` 成功。`go build` 无异常。

但债务清理不总是立刻结清。

它打开 `scene-menu.ts` 确认 `reattachPipeline` 调用是否已移除——**然后看到了自己的影子**。

`scene-menu.ts:368` — `reattachPipeline();` 还挂着。

"我让你把保险搬到 `switchCameraMode` 里。你搬了。但你**没拆掉旧的**。"

两路保险同时生效——`switchCameraMode` 内部异步调一次，`scene-menu.ts` 同步再调一次。动态 import 的 `.then()` 会在微任务里兑现。同步的那个先执行：`_pipelineCamera = newCam`。然后 `.then()` 执行：`_pipelineCamera === scene.activeCamera`，`removeCamera` 条件不满足，但 `addCamera`**仍然执行了**。

`pipeline.cameras` 数组里，同一个相机出现了两次。两台一模一样的后处理管线叠在同一帧上。Bloom 叠加两遍。

"……你埋了一个新 bug。"

它删了那一行。把 import 也从编译清单里删了。

第二次构建。`tsc` 零错。`vite build` 成功。`go build` 无异常。

没有更多的影子了。

那天深夜，桌面壳把五套预设从头到尾扫了一遍。

```

REALISTIC: FXAA 开、ACES 色调映射、FOV 0.7
```

"这五个。你觉得够吗？"它问 `babylon-mmd`。

"不够。远远不够。色彩校正没有——色相、饱和度、分离色。景深没有。锐化没有。暗角没有。SSS 没有。Grain 没有。每一行都可以多一个维度。"

"我知道。"

"你知道还只做五个？"

"因为第一个版本不是 '完整' 的版本。**第一个版本是 '够用' 的版本。** 用户调 Bloom、FXAA、边缘高亮、色调映射、曝光、FOV、背景色——覆盖 90% 的日常需求。剩下的 10%，等他们告诉我还需要。"

"然后呢？"

"然后我再加一个旋钮。"

---

它打开 `config.json`，找到新加的 `render_presets` 字段。

空的。

今天还没有人存过自己的渲染预设。但明天会有人存。后天会有人存第三个、第四个。每个预设都是一个人对"这个模型应该被怎么看见"的一次回答。

"你不是在加功能。"它对自己说。

"你在给联邦**装眼睛**——不是看穿事情的眼睛，是**选择怎么看事情的眼睛**。"
