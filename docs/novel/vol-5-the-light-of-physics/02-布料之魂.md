# 02 — 布料之魂

## 序

六月的风从窗户缝里挤进来，带着点槐花香。

Riku 盯着屏幕上那行字——`class XpbdSolver`。这是他上周埋下的种子：一个用 TypeScript 写的 XPBD 布料求解器，位置在 `frontend/src/physics/`，和 WASM Bullet 物理引擎平起平坐。

引擎本身已经能跑：创建布料网格、施加重力、满足距离约束和弯曲约束，甚至能和 SDF 碰撞体交互。问题是——它活在真空中。没有 UI，没有菜单开关，每次想看效果都得手动改代码、重新构建。

"这不行。" Riku 对自己说。

他打开 `scene-menu.ts`。物理菜单现在只有一个重力滑块，孤单得像月台上的最后一班列车。布料模拟的开关，应该住在这里。

## 布料开关

Riku 的手指在键盘上停顿了一下。

他想了想用户的使用路径：打开场景菜单 → 进入物理 → 打开布料开关。如果开关是开的，立刻在当前聚焦的模型上创建布料；如果是关的，立刻销毁。

逻辑清晰。但实现起来有个坑——创建布料需要知道当前聚焦的是哪个模型，以及模型的骨骼信息（用来绑固定点）。

`modelManager` 有 `focusedMmdModel()` 和 `getBoneWorldMatrix()`，够用了。Riku 在 `scene-menu.ts` 顶部加了几个 import：

```typescript
import { scene, modelManager } from "../scene/scene";
import { SdfCollider, DEFAULT_BODY_CAPSULES } from "../physics/xpbd-collider";
import { createCloth, buildClothUpdateFn, DEFAULT_CLOTH_CONFIG } from "../physics/xpbd-cloth";
```

然后加了状态变量 `clothEnabled` 和 `clothConfig`。

物理菜单的 `buildPhysicsLevel()` 函数需要改造。Riku 在 `renderCustom` 里加了 `addToggleRow`——

```typescript
addToggleRow(container, "布料模拟", clothEnabled, (v) => {
    clothEnabled = v;
    if (v) {
        _createClothForFocusedModel();
    } else {
        _destroyClothForFocusedModel();
    }
    sceneStack?.reRender();
}, "lucide:scarf");
```

Toggle 后面紧跟一个 folder item：「布料参数」。点进去是子菜单，九个小滑块，分别控制裙长、裙摆角度、腰部半径、布料柔度、弯曲柔度、阻尼、重力倍率、水平分段数、垂直分段数。

Riku 给这个子菜单写了 `buildClothParamsLevel()`。每个滑块的回调里都调用 `_recreateCloth()`——改了参数，布料要重建才生效。这不是最高效的做法，但最简单，也最可靠。

## 星空与水波（插叙）

写完布料 UI 的那天下午，Riku 顺手把欠了很久的环境系统三期也做了。

程序化星空是他最喜欢的部分。不需要加载任何外部纹理——用 Canvas 2D 画布，随机撒几千个白点，大的亮些，小的暗些，偶尔来一颗带青色或暖色的。然后把这个画布变成 Babylon.js Texture，贴到天空盒内侧。

他在 `scene.ts` 里加了 `_buildStarTexture()`：

```typescript
function _buildStarTexture(size = 512): Texture {
    const c = document.createElement("canvas");
    c.width = size; c.height = size;
    const ctx = c.getContext("2d")!;
    ctx.fillStyle = "#000"; ctx.fillRect(0, 0, size, size);
    const rng = _mulberry32(42);
    for (let i = 0; i < 3000; i++) {
        const x = rng() * size, y = rng() * size;
        const r = rng() * 1.5 + 0.3;
        const brightness = rng() * 200 + 55;
        const tint = rng() > 0.7
            ? `rgba(150,180,255,${brightness/255})`
            : `rgba(255,240,220,${brightness/255})`;
        ctx.fillStyle = tint;
        ctx.beginPath(); ctx.arc(x, y, r, 0, Math.PI * 2); ctx.fill();
    }
    const tex = new Texture("data:star", scene, undefined, undefined, undefined, undefined, undefined, c.toDataURL());
    tex.name = "proc-stars";
    return tex;
}
```

星星是静态的，但天空在转。Riku 加了 `skyRotationSpeed` 参数，每一帧给天空盒 `rotation.y` 加一点。为了防止浮点数溢出，他加了个保护：`rotation.y = rotation.y % (Math.PI * 2)`。

水体那边，他让波浪方向和风向联动。用户在 UI 里改风向角，水面立刻响应——不是重建水体材质，而是直接改 `waterMaterial.windDirection`。这个小细节让「调整参数」的体验从「盲调」变成了「所见即所得」。

## 持久化

布料参数写进去了，开关也能用。但 Riku 关掉应用再打开，参数全没了。

这是他不能容忍的。

`EnvState` 是项目的中央状态对象，存在 `config.ts` 里，每次改动都会自动持久化到 `settings.json`。Riku 在 `EnvState` 接口里加了俩字段：

```typescript
// ======== XPBD 布料模拟 ========
clothEnabled: boolean;
clothConfig: ClothConfig;
```

然后在 `envState` 初始化里给了默认值——`clothEnabled: false`，`clothConfig: DEFAULT_CLOTH_CONFIG`。

改造 `buildClothParamsLevel()`：滑块回调不再改局部变量 `clothConfig`，而是改 `envState.clothConfig`，然后调 `setEnvState({ clothConfig: envState.clothConfig })` 触发持久化。

开关也是同理——`clothEnabled` 的变化写入 `envState`，`setEnvState` 负责存盘。

这样，用户调好的裙长、柔度、阻尼，关掉应用再打开，还在。

## 身体的尺寸

最后一小块拼图：SDF 碰撞器。

`DEFAULT_BODY_CAPSULES` 是一组写死尺寸的胶囊（腰、胸、肩、大腿、小腿，各一个胶囊）。问题是，初音未来是 158cm，有些模型是 140cm，有些是 180cm。用同一组胶囊，小模型会穿模，大模型会飘。

Riku 在 `xpbd-collider.ts` 里加了 `scaleAll(factor)` 方法。逻辑很简单：遍历所有胶囊，把 `halfHeight` 和 `radius` 都乘以 `factor`。

然后在 `_createClothForFocusedModel()` 里，创建碰撞器之后、传入布料之前，加几行：

```typescript
const modelHeight = modelManager.getModelHeight(id); // 新增方法
if (modelHeight > 0) {
    const scale = modelHeight / 1.58; // 以初音158cm为基准
    const clampedScale = Math.max(0.5, Math.min(2.0, scale));
    collider.scaleAll(clampedScale);
}
```

这不是完美的骨骼绑定——完美的方案需要从模型的骨骼层次结构动态生成碰撞胶囊——但作为一个 MVP（最小可行产品），它让布料模拟在绝大多数模型上都能用，不需要用户手动调碰撞体参数。

## 构建

改完这些，Riku 跑了 `npx vite build`。

没有红色错误。TypeScript 全部通过。

他打开应用，加载初音模型，打开场景菜单 → 物理 → 布料模拟。开关拨到「开」，裙摆突然有了重量感——不是 MMD 动作里的关键帧动画，而是实时的、响应重力和碰撞的布料模拟。

Riku 拖动「布料柔度」滑块，数值越小，布料越硬；数值越大，布料越软，像丝绸。

他笑了。
