# 云层协议

## 楔子

凌晨一点四十七分。

MikuMikuAR 联邦的第七十二次架构会议，在所有人都不情愿的深夜中开始了。会议室的光线调到最低档，屏幕上只有一块裸露的灰色球体。

球体上什么都没有。

"这是在渲染云层？"首席产品经理陈眠打了个哈欠。

"这是渲染失败。"Riku 把控制台的日志拍到大屏幕上，全屏都是红色的 `isReady: false`。

"三个方案。死了一个，一个半死不活，第三个还没生出来。"

他顿了顿。

"我们今天晚上就在这儿，把云层协议写到能看为止。"

---

## 第一章：后处理陷阱

如果时间倒回六个小时，Riku 还信心满满。

"PostProcess，"他在白板上画了一条横线，"全屏四边形，片段着色器里走 Ray Marching。Vite 不干涉，Babylon 不报错，Wails 不崩溃。完美。"

他花了十五分钟写完着色器。一百二十行 FBM 噪声，Henyey-Greenstein 相位函数，双散射近似。编译。构建。运行。

屏幕全白了。

不是云的那种白——是"你的应用死了"的那种白。

"alpha 混合没开。"他冷静地加上 `alphaMode = 0`。构建。运行。

全白。

"手动混合。"他在着色器里采样 `textureSampler`，手动计算 `scene.rgb * (1-alpha) + cloud.rgb * alpha`。构建。运行。

还是全白。而且这次连按钮都被白色淹没了，UI 像是沉入乳白色海洋的沉船残骸。

"别试了，"测试部的老林端着咖啡走进来，"PostProcess 路径废弃。DefaultRenderingPipeline 加 WebView2 的兼容坑，你挖不完的。"

Riku 盯着屏幕看了十秒，删掉了 `PostProcess` 的整个文件。

"联邦宪法第一条，"他说，"止损要及时。"

---

## 第二章：球体内部的宇宙

下一个方案来得很快——ShaderMaterial + 球体。

"用 MeshBuilder.CreateSphere，直径八百，双面渲染，摄像机在球体内部看云层。"

"你确定？"陈眠问。

"确定。球体内表面当画布，Ray Marching 的方向从摄像机指向球面顶点。物理上是对的。"

十分钟后，控制台输出了一行好消息：

```
[VolCloud] ShaderMaterial isReady: true
```

然后他们看到了一个半透明的、均匀的灰色球壳。

"这不是云，"老林说，"这是一个雾气球。"

"我知道这不是云。"

"看起来像你用一个灰色马克笔在天空画了个圈。"

"我说了我知道！"

Riku 深吸一口气，开始逐行检查着色器。

第一个问题在第三行就抓到了——顶点着色器里 `vWorldPos = position`，传递的是本地坐标而非世界坐标。摄像机位置是世界的，射线方向 `normalize(vWorldPos - cameraPosition)` 是完全错误的。

"世界矩阵。"他加了一行 `world * vec4(position, 1.0)`。

第二个问题：`cloudBaseY = 395`，硬编码的高度。摄像机在 Y=5 的地面，云层在 400 米高空，射线走 144 单位就停了，根本碰不到云。

"步长 3.0，步数 48，最大距离 800。"他改了参数，"覆盖距离=144，不够。步长 6.0，步数 80，最大距离 600。覆盖 480。还不够。"

"步长 10，步数 200，最大距离 3000。够不够？"

"总够。但性能会炸。"

"先让它能看，再让它快。"

云层终于出现了——但是一整块巨大的、覆盖整个天空的白色棉被。

"太大了。"陈眠说。

Riku 把噪声频率从 `0.006` 提到 `0.015`。云团缩小了一点，但球体本身像一个巨大的月亮挂在视野里，怎么调都是错的。

"问题不是频率，"他突然明白了，"是球体本身在渲染。低密度区域的 alpha 太低，但片段没有 discard，整个球体都是可见的。"

他加了一行：

```glsl
if (gl_FragColor.a < 0.02) discard;
```

球体消失了。云朵留了下来。

会议室安静了两秒。然后老林吹了声口哨。

---

## 第三章：蓝色幽灵

云朵有了，但有些地方不太对劲。

"看到那些蓝色的小点了吗？"陈眠把屏幕放大，"云朵边缘，稀薄的地方，透着蓝色的光。"

"那是背景色。alpha 混合没正确启用。"

"`needAlphaBlending: true` 已经设了。"

Riku 想了想，打开 Babylon.js 的源码开始查。三分钟后他骂了一句脏话。

"ShaderMaterial 的 `needAlphaBlending` 只是建议，默认透明模式是 `ALPHA_DISABLE`。渲染器直接忽略了 alpha 输出。"

他加上一行：

```typescript
mat.transparencyMode = 2; // ALPHA_BLEND
```

蓝色小点消失了。但云层内部的白色依然透着淡蓝——光学深度累积不够，背景色还是穿了过来。

"方案 B，"Riku 说，"改为累积光学深度。不是每次 exp(-常数)，是真的走一步算一步。"

他在循环里添加了 `opticalDepth += d * dynamicStep * 0.12`，用 `T = exp(-opticalDepth)` 替代了原来的经验公式。

云层的核心变成了纯白。边缘保持半透明。背景色被彻底挡在外面。

"这才像云。"陈眠说。

---

## 第四章：Silver Lining

"还缺一样东西。"

Riku 在代码末尾插入了十行着色器代码。

"边缘辉光。Silver Lining。阳光穿过云层边缘时产生的暖金色发光。"

他解释了原理：在透光率 `T` 的半透明边缘（`T ≈ 0.3~0.7`）达到峰值，且只在背光面出现。用 `T * (1 - T) * 4` 计算强度曲线，再用背散射角度因子限制方向。

编译。运行。

云层的边缘亮起了一圈淡淡的金色。

"我靠。"老林的咖啡杯停在了半空。

"好看吧。"

"不是好看——是真的像云了。像日落时的那种云。"

Riku 盯着屏幕看了很久。这是他今天第一次觉得——这个方向是对的。

---

## 第五章：联觉

凌晨零点四十分，风还不对。

"风打开了，但云层没动。"陈眠指了指 UI 上的风力开关。

Riku 检查了参数。`windDirection` 乘以 `0.005`——这是为旧版平面云的纹理 UV 偏移设计的系数。对三维世界空间的噪声来说，这个值小到可以忽略不计。

"0.005 是给纹理漂移的。3D 云层需要世界空间的偏移量，系数应该差两个数量级。"

他把系数从 `0.005` 提升到 `2.0`。

云层开始缓缓流动。

"还不够快。"

"着色器里还有一道 `* 0.03` 的限制。"

"去掉？"

"去掉。"

云层像被解开了缰绳，开始在天幕上从容地漂移。背光面的边缘辉光随着云的移动缓缓变化，像是天空在呼吸。

"现在的问题是——"Riku 说到一半停住了。

"什么？"

"场景光。方向光的方向和颜色都是硬编码的。云层的明暗不随太阳位置变化。"

他加了两个 uniform：`sceneLightDir` 和 `sceneLightColor`。每帧从场景的 `DirectionalLight` 同步。

"现在你调方向光的角度，云层的明暗和辉光位置会跟着走。"

陈眠拖了一下方向光的滑块。屏幕上的云背光面暗了下去，迎光面亮了起来，边缘的金色辉光像被点燃了一样。

"漂亮。这他妈的是真的漂亮。"

---

## 终章：收敛

凌晨一点四十六分，Riku 删掉了最后一段死代码。

"提交了。"

"等一下，"陈眠说，"给它起个名字。"

"什么？"

"这次提交。你改了这么多东西，总得有个名字吧。"

Riku 想了想，在提交信息里打了一行字：

```
feat: volumetric cloud (ShaderMaterial-on-Sphere) with Ray Marching
```

然后他在括号里补了一行小字：

```
——从抹布到棉花糖的进化史。
```

Push 失败了。SSL 证书验证报错，token 权限不够，git 被网络层面的各种问题堵住，死活发不出去。

"算了，"Riku 关掉终端，"云层在那里就行了。Git 推不推也无所谓了。"

陈眠看了一眼窗外。凌晨的街道安静得像一幅画。如果仔细看，能看见天空的最东边有一层薄薄的云——不是程序生成的，是今天刚好有的。

"你知道吗，"他说，"我们调了一晚上的云层参数，结果今晚真的有一片云挂在那儿。"

"巧合。"

"也是这行的浪漫。"

Riku 没有回答。他已经在看明天的待办清单了。但在合上电脑之前，他忍不住又看了一眼那片云。

它的颜色、厚度、边缘的光——和屏幕上的一模一样。

---

*「联邦的云层，终于像云了。」*

*—— MikuMikuAR 开发日志，2026 年 6 月 29 日*
