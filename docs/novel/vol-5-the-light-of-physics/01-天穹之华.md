# 天穹之华

> 第五卷 · 物理之光 · 第一章
>
> 对应真实事件：环境系统三期扩展（太阳光晕、时间流动画、天空预设 4→7、水体动画速度、水下后处理、Perlin 噪声、程序化星空、天空旋转、水面预设）

---

联邦的穹顶已经存在了很久。从第三卷第九章「天穹之境」开始，联邦就有了天空、地面、粒子、风、云、雾。它工作，但它是静态的——天空永远是那三种颜色，水面永远是你调好时的样子，太阳永远悬在固定角度。

这不是穹顶的错。是联邦还没学会让穹顶「活」起来。

## 一、对标的镜子

DanceXR——联邦北方的强邻，它的穹顶会动。太阳会升起落下，星空会在深夜出现，水面会有风浪。联邦的议会成员在 DanceXR 的穹顶下站了十分钟，然后回到自己的代码前。

「我们的穹顶是死的。」

这句话很难说出口。联邦花了整整一卷（第三卷）才建成穹顶系统，现在有人告诉它：你建了一座静止的博物馆。

对标文档（`docs/roadmap.md`）里，环境系统的完成率是 100%。但这是「功能完成率」——有天空、有水面、有粒子，不等于这些功能达到了 DanceXR 的水准。

议会首席在白板上画了一条线：

```
功能完成率：100%
```

「我们需要补上那 40%。」

## 二、第一期：日光与时间的魔法

第一期从太阳开始。

联邦的天空渐变纹理是用 Canvas 2D 画的——`_buildGradientTexture` 函数，256×256 的 Canvas，按 `skyColorTop`、`skyColorMid`、`skyColorBot` 三个颜色画径向渐变。这个函数已经存在了，但它画出来的太阳是「平的」——一个圆盘，没有光晕，没有光芒四射的感觉。

「我们想要的是太阳光晕。」

太阳光晕——在真实世界里，你看向太阳时，它周围有一圈光晕，因为大气散射。在渲染里，这可以通过在太阳方向叠加一个径向渐变来实现。

议会成员打开了 `scene-env-impl.ts`，找到 `_buildGradientTexture`。

原来的代码：
```typescript
const cx = 128, cy = 64; // 太阳位置固定
// 画渐变
```

新的代码：
```typescript
const sunAngle = envState.skyRotationY; // 太阳角度
const sunX = 128 + Math.cos(sunAngle) * 64;
const sunY = 64 + Math.sin(sunAngle) * 32;
// 在太阳位置画光晕
const haloRadius = 30;
const gradient = ctx.createRadialGradient(sunX, sunY, 0, sunX, sunY, haloRadius);
gradient.addColorStop(0, `rgba(255, 255, 200, 0.8)`);
gradient.addColorStop(0.5, `rgba(255, 200, 100, 0.3)`);
gradient.addColorStop(1, `rgba(255, 150, 50, 0)`);
ctx.fillStyle = gradient;
ctx.fillRect(0, 0, 256, 256);
```

太阳光晕出现了。它不大，但它在那里——太阳不再是一个平面圆盘，它有光。

然后是第一期第二个功能：时间流动画。

「时间流」——让太阳自动在天空中移动，模拟一天 24 小时。这是一个经典的渲染功能，所有开放世界游戏都有。

实现很简单：
1. 添加 `startTimeOfDay()` 函数，启动一个 `setInterval`，每 100 毫秒更新 `sunAngle`
2. `sunAngle` 改变 → 太阳位置改变 → 光晕位置改变 → 天空纹理需要重建
3. 用 `_gradientCache` 缓存不同 `sunAngle` 的天空纹理

但「很简单」的实现隐藏着性能陷阱——如果每 100 毫秒都重建天空纹理，性能会崩。

议会的运行委员会站了出来：「我们用缓存。`_gradientCache` 已经存在了，我们只需要把 `sunAngle` 加入缓存键。」

```
缓存键 = `${topR},${topG},${topB},${midR},${midG},${midB},${botR},${botG},${botB},${brightness},${sunAngle},${starsEnabled}`
```

`sunAngle` 改变 → 查找缓存 → 命中 → 直接返回纹理（0 毫秒）
`sunAngle` 改变 → 查找缓存 → 未命中 → 重建纹理（200 毫秒）→ 存入缓存

时间流动画跑起来了。太阳在天空中缓慢移动，光晕跟着动，天空颜色随着太阳角度变化。

联邦的穹顶第一次「活」了。

## 三、第二期：从四个预设到七个

第二期的任务是扩展天空预设。

联邦的天空预设原来有四个：清晨、正午、黄昏、夜晚。`applyEnvPreset` 函数里硬编码了四个预设的颜色值。DanceXR 有七个。

「加三个：黎明、薄暮、午夜。」

这看起来是一个简单的任务——复制粘贴，改几个颜色值。但议会成员在改代码时发现了问题：

「我们的预设是硬编码的。如果以后要加第八个、第九个预设怎么办？」

更好的方案是：把预设存成数据结构，而不是硬编码在 `applyEnvPreset` 里。

```typescript
interface EnvPreset {
    name: string;
    skyColorTop: [number, number, number];
    skyColorMid: [number, number, number];
    skyColorBot: [number, number, number];
    // ... 其他参数
}

const ENV_PRESETS: EnvPreset[] = [
    { name: "清晨", skyColorTop: [0.3, 0.5, 0.8], ... },
    { name: "正午", skyColorTop: [0.5, 0.7, 1.0], ... },
    // ...
];
```

然后 `applyEnvPreset` 变成：

```typescript
export function applyEnvPreset(presetName: string): void {
    const preset = ENV_PRESETS.find(p => p.name === presetName);
    if (preset) {
        setEnvState({ ...preset });
    }
}
```

更灵活，更易扩展。

然后第二期加入了 `.env` 格式的导入导出——让用户可以把自己的环境配置保存成文件，分享给其他人。

`.env` 文件格式：
```json
{
    "name": "我的日落",
    "skyColorTop": [0.8, 0.4, 0.2],
    "skyColorMid": [0.9, 0.6, 0.3],
    "skyColorBot": [0.2, 0.1, 0.1],
    // ...
}
```

`exportEnvPreset()` → 把当前 `envState` 序列化成 `.env` 文件
`importEnvPreset()` → 读取 `.env` 文件，应用到 `envState`

联邦的穹顶配置第一次可以「带走」了。

## 四、第二期（续）：水下的世界

第二期的后半部分是水体增强。

联邦的水面是用 `WaterMaterial`（来自 `@babylonjs/materials`）实现的。`WaterMaterial` 是一个强大的水面着色器——它支持反射、折射、波浪动画、法线贴图。

但联邦的水面有几个问题：
1. 动画速度不能调——波浪永远以固定速度动
2. 没有水下视角——相机入水后，看到的还是水面上的视角
3. 水面是平的——没有凹凸纹理

第二期逐个解决。

**水体动画速度控制**：

`WaterMaterial` 有 `windForce` 和 `waveSpeed` 两个属性，控制波浪的强度和速度。原来联邦没有暴露这两个属性给用户。

解决方法：在环境菜单添加滑块，控制 `waterAnimSpeed`（0.1~5 倍速）。

```typescript
// env-menu.ts
addSliderRow(container, "动画速度", envState.waterAnimSpeed ?? 1, 0.1, 5, 0.1, (v) => {
    setEnvState({ waterAnimSpeed: v });
    if (waterMaterial) {
        waterMaterial.windForce = v;
        waterMaterial.waveSpeed = v;
    }
}, "lucide:fast-forward");
```

**水下后处理**：

这是最酷的功能。当相机入水后（`camera.position.y < waterLevel`），联邦自动切换后处理管线——添加蓝色色调、降低清晰度、添加雾效，模拟水下视角。

实现：
1. 在渲染循环中检测相机高度
2. 如果 `camera.position.y < waterLevel` 且还没启用水下效果 → 添加水下后处理
3. 如果 `camera.position.y >= waterLevel` 且已启用水下效果 → 移除水下后处理

水下后处理包含：
- 蓝色色调（`ImageProcessingConfiguration.toneMappingEnabled`）
- 雾效增强（`scene.fogMode = FOGMODE_EXP2`, `scene.fogDensity` 增大）
- 色差效果（`ChromaticAberrationPostProcess`）模拟水下折射

**Perlin 噪声凹凸纹理**：

平面的水面看起来不真实。真实的水面有细微的凹凸——波浪、涟漪。

解决方法：用 Perlin 噪声生成法线贴图。`perlin2` 函数在第二期构建优化时已经被缓存了，现在可以直接用。

```typescript
function _generateWaterBumpTexture(size: number = 512): DynamicTexture {
    const texture = new DynamicTexture("waterBump", size, scene, true);
    const ctx = texture.getContext();
    
    for (let y = 0; y < size; y++) {
        for (let x = 0; x < size; x++) {
            const noise = perlin2(x * 0.02, y * 0.02); // Perlin 噪声
            const pixel = Math.floor((noise + 1) * 128); // 映射到 0~255
            ctx.fillStyle = `rgb(${pixel}, ${pixel}, ${pixel})`;
            ctx.fillRect(x, y, 1, 1);
        }
    }
    
    texture.update();
    return texture;
}
```

然后把这张噪声图设置成 `WaterMaterial.bumpTexture`。

水面不再平坦。它有细微的凹凸，光在水面上反射时会产生微妙的变化。

## 五、第三期：星空与旋转

第三期是装饰性功能的狂欢。

**程序化星空**：

深夜的天空应该是黑色的，但联邦的深夜天空是深蓝色的。没有星星。

「我们想要星空。」

程序化星空——用 Canvas 2D 在天空纹理上画星星。不需要外部纹理文件，不需要加载图片，纯代码生成。

算法：
1. 在 512×256 的 Canvas 上随机生成 200 个点
2. 每个点是一个星星——画一个白色的圆，半径 0.5~2 像素
3. 部分星星添加十字光芒（画一个「+」形）
4. 把 Canvas 存成纹理，应用到天空盒

```typescript
function _addStarsToSkyTexture(ctx: CanvasRenderingContext2D, width: number, height: number): void {
    const starCount = 200;
    for (let i = 0; i < starCount; i++) {
        const x = Math.random() * width;
        const y = Math.random() * height * 0.5; // 星星只在天空上半部分
        const radius = Math.random() * 1.5 + 0.5;
        
        ctx.fillStyle = `rgba(255, 255, 255, ${Math.random() * 0.5 + 0.5})`;
        ctx.beginPath();
        ctx.arc(x, y, radius, 0, Math.PI * 2);
        ctx.fill();
        
        // 十字光芒
        if (Math.random() > 0.7) {
            ctx.strokeStyle = `rgba(255, 255, 255, 0.3)`;
            ctx.lineWidth = 0.5;
            ctx.beginPath();
            ctx.moveTo(x - 3, y);
            ctx.lineTo(x + 3, y);
            ctx.moveTo(x, y - 3);
            ctx.lineTo(x, y + 3);
            ctx.stroke();
        }
    }
}
```

星空出现了。当 `starsEnabled` 开启、太阳角度在深夜范围时，星空自动出现在天空纹理上。

**天空旋转动画**：

第一期的时间流动画让太阳移动。第三期让整个天空旋转——天空盒缓慢旋转，云慢慢飘过，天空纹理慢慢转动。

实现：在渲染循环中每帧更新天空盒的 `rotation.y`。

```typescript
let skyRotationSpeed = 0; // 度/秒
// 在渲染循环中：
if (skyRotationSpeed > 0) {
    skyBox.rotation.y += skyRotationSpeed * deltaTime;
}
```

然后在环境菜单添加滑块，控制 `skyRotationSpeed`（0~5 度/秒）。

**波浪-风向联动**：

联邦的风系统（`windEnabled`, `windDirection`, `windSpeed`）原来只影响粒子系统。水面不知道有风。

第三期把风和波浪联动起来——风的方向影响波浪的方向，风速影响波浪的高度。

```typescript
// 当 windDirection 或 windSpeed 改变时
if (waterMaterial) {
    waterMaterial.windDirection = new Vector2(windDirection[0], windDirection[2]);
    waterMaterial.windForce = windSpeed;
}
```

**水面预设**：

像天空预设一样，联邦现在也有了水面预设。五种：
1. 平静（calm）——无风，水面如镜
2. 微澜（ripple）——微风，细小波浪
3. 海浪（ocean）——强风，大浪
4. 风暴（storm）——狂风，巨浪
5. 热带（tropical）——温暖，中等波浪

每种预设预配置了 `windForce`、`waveSpeed`、`waterColor`、`waterTransparency`。

## 七、散场

三期扩展的技术细节很复杂——Canvas 2D 渐变、Perlin 噪声、后处理管线、纹理缓存、时间动画。但最核心的改变不是技术，是认知：

**联邦的穹顶不再是「设置一次就用一辈子」的静态配置。它是动态的、有时间的、有生命的。**

太阳光晕跟着太阳走。时间流动画让太阳在天空移动。星空在深夜出现。水面跟着风起浪。

这不是功能的堆砌。这是联邦的穹顶学会了「变化」。

> **静止是死亡的开始。变化是生命的证据。**
