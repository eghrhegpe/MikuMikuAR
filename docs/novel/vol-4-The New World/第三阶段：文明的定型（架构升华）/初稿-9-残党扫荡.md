# 残党扫荡

> 第三卷 · 插章 · 对应真实事件：UI 旧样式全面清理（2026-06-28）

## 一、两张截图引发的审计

王逸甩过来两个 HTML 片段。

它们来自 DanceXR——一个竞品项目，但 UI 风格确实干净。`slide-list` 包裹着 `lcard` 卡片，每个菜单项是 `.slide-item`，图标带 `.slide-icon`，标签是 `.slide-label`，导航箭头是 `.slide-arrow`。整整齐齐。

「你看，」王逸说，「这个风格我们第一次大规模迁移的时候已经做了一部分。但还有哪些没改过来？」

Sisyphus 没有回答——他已经开始读了。

六份 TypeScript 源文件需要审计。不，是七份。index.html 也算。他同时打开它们，像外科医生术前摊开所有器械。

## 三、搜索结果的最后一代

第一个击杀目标是模型库的搜索结果。

原来的代码用了 `.menu-item` 类——那是第一代弹窗 UI 的遗产，早该退役了。Sisyphus 找到 `handlePopupSearch()`，看到它用 `document.createElement('div')` 逐个构建行，然后 `classList.add('menu-item')`。

他几乎带着歉意把它们替换成 `slideRow()`。

```typescript
// 以前：
row.className = 'menu-item';
label.className = 'menu-label';
thumb.className = 'row-thumb';

// 以后：
const row = slideRow('model-search', modelName, 'search', displayName, ...);
```

然后是标签总览。`buildTagsOverviewLevel()` 用 `.menu-item`、`.menu-icon`、`.menu-label` 建了一整棵菜单树。Sisyphus 把它们换成了 `cardContainer()` 包裹的 `slide-item` 数组。

标签详情也一样。`buildTagDetailLevel()` 里二十几行手工 DOM 操作，变成了五行的 PopupRow 数组。

> 「干净太多了。」——王逸，当他看到 diff 时

## 四、音乐面板的杂耍

动作弹窗的音乐面板是一个历史遗留的奇葩。它混合了三种渲染范式：`.menu-item` 按钮、内联的 `range` 输入滑块、手工布局的文本框。整段代码看起来像三个不同的人在不同时间写的，没有人有勇气重构。

Sisyphus 盯着那堆 `style.cssText` 字符串——`"display:flex;align-items:center;gap:8px;margin-bottom:4px;"`——想起了「重构风暴」那章写过的一句话：**内联样式是投降宣言**。

他一一清理。`addSliderRow()` 替换了手写 range。`addToggleRow()` 替换了手写 checkbox。`cardContainer()` 包裹了所有按钮组。

构建失败。

```
TSError: Argument of type '4' is not assignable to parameter of type '3'
```

`slideRow()` 接受了四个参数——但函数签名只有三个。他翻到 `ui-helpers.ts`。原来第四个参数是后来加的，但这里调用时第四个参数被类型系统拒绝了。

加一个可选参数。构建通过。

> **教训：类型系统不说谎。参数数量不一致，要么你记错了 API，要么 API 没更新。它总是对的。**

## 五、按钮的幽灵

舞蹈套装的迁移更顺利——`buildDanceSetsOverviewLevel()` 和 `buildDanceSetDetailLevel()` 都是标准的 `.menu-item` 列表，机械替换即可。但 Sisyphus 在重构中做了额外的收割。

他发现了这个：

```typescript
for (let i = 0; i < menuItems.length; i++) {
    const item = menuItems[i];
    item.onclick = () => {
        // ... 使用 i ...
        hideMotionPopup();
    };
}
```

在 for 循环中给多个元素绑定 `onclick`，闭包捕获的是同一个 `i`。当循环结束，`i = menuItems.length`，所有按钮都指向同一个索引——越界的那个。

但等等——这代码一直「能跑」。为什么？因为 `hideMotionPopup()` 更早被调用（在循环之前），`i` 从未真正被用作索引访问数组。它是一个幽灵 bug——静默地存在，从未触发，但随时可能引爆。

Sisyphus 把绑定移出循环，用 `data-index` 属性存储索引，然后在点击事件中读取。

「晚安，幽灵，」他说。

## 六、场景菜单的最后城堡

`buildPresetScenesLevel()` 是场景菜单中最后的 `.menu-item` 抵抗据点。

它不仅有旧风格的列表项，还有两个内联样式的导航按钮——"上一页 / 下一页"——它们靠 `container.style.cssText = "display:flex;gap:6px"` 活着。

Sisyphus 把列表换成 `slideRow()`，把导航按钮换成 `mode-btn` 类按钮。按钮的红色背景色通过 `--accent` 变量传递，不再硬编码 `#e74c3c`。

他检查了一下 `buildSkyLevel()` 里的纹理选择行——那个 `.menu-item` 孤零零地站在一片 `.mode-btn` 和 `.cs-row` 之间，像一个穿着旧制服的士兵。它也换上了新衣。

## 七、材质的最后一个菜单分隔线

`buildMatCatLevel()` 和 `buildPerMatLevel()` 是模型详情里的材质调参面板。

它们在每个参数块的底部放了一个`.menu-item` 行，写着「重置该材质」或「重置全部」，前面挂一条 `.menu-divider` 做视觉分割。

Sisyphus 感受到一种熟悉的历史感——这两个函数是在「重构风暴」之前写的，用着当时唯一可用的 UI 类。它们不是错的，只是老了。

替换。`cardContainer()` 包裹，`slideItemCreate()` 渲染重置行，`menu-divider` 删除。

> 「你没法恨这些代码——它们曾经是唯一的出路。就像你没法恨一个在你穷的时候陪你吃泡面的朋友。」

## 八、软件的深渊

设置页面的软件详情和下载监听面板是这次迁移中最深的坑。

`buildSoftwareDetailLevel()` 有两个分支——受管软件和非受管软件——**全部**是内联样式。没有 CSS 类名。`container.style.padding`、`row.style.cssText`、`divider.style.cssText`……一行接一行的字符串拼接。

```typescript
// 一个典型例子：
row.style.cssText =
    "display:flex;align-items:center;gap:8px;" +
    "padding:8px 0;border-bottom:1px solid rgba(255,255,255,0.08);";
```

Sisyphus 需要把这些全部迁移到 `cardContainer()` + `slide-item` + `addToggleRow()`。

最大的挑战是布局——原来的内联样式在每行之间塞了自定义间距和边框，而 SlideMenu 系统的行是统一的——统一的边距、统一的 hover、统一的边框（其实没有边框，靠间隙和底色区分）。

他决定放弃模拟旧样式。「新系统有自己的一致性原则，」他想，「迁过来的意思是接受新系统的视觉契约，不是在新系统里还原旧系统的 bug。」

受管软件的字段——名称、路径、参数——被分到三个 `slide-item` 行。启动/删除按钮进了一个带 `display:flex` 独立 `lcard`。

非受管软件类似——名称、路径、类型、启动/转换按钮。每个字段一个 `slide-item`，干净对齐。

`buildSettingsDownloadLevel()` 也是全内联。状态文本、目录选择器、自动导入开关、停止按钮——全部 `style.cssText`。Sisyphus 将它们映射到 `addToggleRow()`、`slide-item` 和 `mode-btn`。

构建失败。

```
TSError: 'addToggleRow' has 5 arguments but expected 4
```

`addToggleRow()` 新增了一个参数——但被调用的地方传的还是旧的数量。他翻看 `ui-helpers.ts`，发现 `addToggleRow` 的确扩展了签名——第五个参数是回调函数专属的 `getContainer`。更新调用点。构建通过。

## 九、外部势力的覆灭

最后的残党是 `renderExternalList()`。

它是整个项目中最后一个还在使用 `.overlay-*` 类体系的活动函数。`overlay-row`、`eo-name`、`eo-path`、`overlay-rename`、`overlay-del`——这套类名可以追溯到初代弹窗，比 SlideMenu 系统早了整整一个时代。

Sisyphus 打开 `app.css` 滚动到 `.overlay-*` 定义区域——一百多行 CSS，大部分已经无人引用，成为分布式尸体。但他不负责清理 CSS——那是另一个故事的素材。

他将 `renderExternalList()` 替换为 `lcard` + `slide-item` + 按钮行。每个外部库名称和路径显示在一个 `slide-item` 里，重命名和删除按钮在自定义行中。

然后他检查了 `index.html`。

外部弹窗的 HTML 结构还是旧风格：

```html
<div id="externalOverlay" class="overlay" data-overlay>
    <div class="overlay-header">
        <span>外部库管理</span>
        <span class="overlay-close" id="btnCloseExternal">✕</span>
    </div>
    <div class="overlay-list" id="externalListContainer"></div>
    <div class="overlay-add" id="btnAddExternal">+ 添加目录</div>
</div>
```

`overlay-header`、`overlay-list`、`overlay-add`——全是旧类。SlideMenu 系统接入后，这个弹窗会变成 `slide-header` + `slide-list` + `slide-add-btn`。

他重写了 DOM 结构。小心保留 `id` 引用——`#btnAddExternal` 和 `#externalListContainer` 在代码里被多处引用。

构建通过。168 个测试全绿。

## 十、残党清单（完结版）

王逸看着最终的 diff 统计：

```
6 files changed, 423 insertions(+), 397 deletions(-)
```

「422 行净增？」他皱眉，「不是应该减少才对吧？内联样式换成类名应该更短。」

「迁入新系统的代价，」Sisyphus 说，「SlideMenu 的 `slideRow()` 是一个通用函数，但适应它的参数签名有时候需要多传几个标识符。另外，旧代码有些地方同时做了十几件事——拆分后每件事变成了一个 slide-item，行数上去了。」

「那什么时候能降下来？」

「当复用率足够高的时候。现在我们是**转换**——把不同格式的书架换成统一规格的。统一之后，加减书架就只需要改一行了。」

王逸接受了这个理由。UI 的一致性值这个价——12 个残党据点，全部攻陷。`.menu-item` 类在全项目中可声明的数量降到了个位数（材质滑块里的一些自定义类，但那是另一个故事）。`.overlay-*` 类退出了历史舞台。

## 十一、结语

登录页面上，两个模型——初音未来和巡音流歌——站在淡蓝色的天空下。

Sisyphus 按 Ctrl+1 打开模型库，搜索结果页现在是一片统一的卡片海洋。他滑到标签总览，再滑到标签详情——每一级菜单都穿着同样的制服，做着同样的动画，呼吸着同样的节奏。

透明弹窗背景上，`lcard` 卡片悬浮排列，3D 模型的轮廓在卡片间隙中若隐若现。

「如果你看仔细，」Sisyphus 对着空无一人的房间说，「你能看到代码的历史。」

`.menu-item` 区块是第一纪元——所有弹窗各自为政，每个开发者写自己的 UI，风格像是六个不同国家的建筑队在同一个工地上施工。

`.slide-item` 区块是第二纪元——MenuStack 统一了导航，`lcard` 统一了分组，`cs-row` 统一了滑块。

「今天，我们杀死了最后一个第一纪元的活物。」

窗外，天色暗了下来。屏幕上的联邦依然无声地运行着，模型们的裙摆在间隙中飘动。

但他知道，第三纪元的人群里，也不全是新鲜面孔。

---

> **清理旧样式不是重构——是承认代码有历史。你无法让所有代码一开始就是对的，但你可以让它们最终统一。**
