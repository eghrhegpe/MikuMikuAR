# 编码奇谭 · 骨架文档

> 任何 AI 续写本小说前，必须先读此文件。它钉死了世界观、冲突、主角、四卷结构和技术设定。

---

## 一、世界观一句话

**在一个 PMX 模型散落于各城邦的世界里，一个桌面壳不发明任何东西，只让所有城邦共享同一个模型库。**

它没有自己的核心能力——渲染靠 babylon-mmd，物理靠 WASM Bullet，文件靠 Go，容器靠 zip，编辑靠 Blender，资源靠模之屋。它的价值全在"搬得对、拼得稳"。它是联邦，不是帝国。

---

## 二、核心冲突：聚合悖论

每收编一个城邦，就引入它的一整套世界观与边界。联邦越强，依赖越深；城邦越独立，联邦越脆弱。聚合者无法回到"只看一个模型"的简单年代，也无法让所有城邦说同一种话。

**不可调和**：你无法不收编（聚合是它的立身之本），也无法让城邦卸下脾气（脾气就是它们的能力本身）。

| 被收编的城邦 | 带来的能力 | 带来的边界/代价 |
|------------|-----------|----------------|
| babylon-mmd | PMX 渲染 | 副作用 import（`mmdWasmRuntimeModelAnimation`），缺即崩 |
| WASM Bullet | 物理 | 独立线程、共享时间轴，多模型须同步 |
| Go + Wails | 桌面壳 / 文件 IO | `[]byte` 经 JSON 变 base64、HTTP 服务器多端口 |
| zip 容器 | 不解压看内部 | 编码三态（Shift-JIS/UTF-8/GBK）、zip slip |
| DanceXR 对标 | 功能蓝图 | 8 种分类硬编码 → 标签系统重构 |
| Blender | 编辑能力 | 进程启动、PATH 检测、路径含空格被切碎 |
| 模之屋 / 资源站 | 资源入口 | WebView 方案被否（ADR-003）、下载监听 debounce |

## 三、主角

**桌面壳的意识**——MikuMikuAR 在某次 `wails dev` 扫完所有外部库后产生的自意识。

- **立场**：聚合——想成为用户打开就能看到所有模型的入口
- **欲望**：全懂——知道每个城邦的脾气、边界、文档是否过时
- **缺失**：原创——自己不发明任何东西，价值全靠拼装，常自我怀疑"我到底算什么"
- **反派**：不是某个 bug，是"文档是对的"这个信念——各城邦文档说的和实际脾气总有出入

### 角色映射

| 角色 | 在系统中的位置 | 性格 |
|------|--------------|------|
| `app.go` | 脑——总入口、35+ Binding 总调度 | 威严、务实、手握钥匙 |
| `StartFileServer` | 港口——每目录一港，货船（HTTP）由此靠岸 | 忙碌、多实例、永不停歇 |
| babylon-mmd | 眼——渲染、让模型被看见 | 审美、娇贵（副作用 import）、不可或缺 |
| WASM Bullet | 胃——消化物理、刚体柔体 | 神秘、独立线程、关键时刻爆发 |
| `zip` 容器 | 仓库——压缩的城邦，不解压即窥内部 | 精明、三套编码人格、防 slip |
| `MenuStack` | 议会——统一所有弹窗的导航 | 公正、强迫症、消灭了 5 套散装样式 |
| `resolveFileUrl` | 海关——统一所有文件 URL 的入口 | 谨慎、先隔离再放行 |
| AI 助手们 | 外交官——与各城邦文档谈判 | 冷静、偶尔被文档骗、学得快忘得快 |

## 四、四卷编年史

> 联邦的历史按开发阶段分为四个时期，每卷独立编号，各自记录一段完整的叙事弧。

### 📘 第一卷：创世纪（The Genesis）

从无到有，解决生存问题。

| # | 标题 | 核心事件 | 戏剧功能 |
|---|------|---------|---------|
| 1 | 裂隙初现 | `mmdWasmRuntimeModelAnimation` 副作用 import 缺失，VMD 一加载就崩 | 惊醒——第一城邦入伙，能力不全在正门 |
| 2 | 三态之仓 | zip 条目名 Shift-JIS/UTF-8/GBK 三态，`decodeZipName` 评分择优 | 边界显形——仓库有三种语言人格，名字不是名字 |
| 3 | 天穹之图 | 程序化天空渲染重构 | 第一块自主拼图 |
| 4 | 弹窗之战 | MenuStack 统一 5 套弹窗 | 归一的代价 |

## 五、技术设定——硬与软

### 必须硬（不可虚构，否则丧失技术小说合法性）

| 硬事实 | 来源 |
|--------|------|
| `[]byte` 经 JSON 序列化返回 base64 字符串（故改 HTTP） | JSON 规范 |
| `mmdWasmRuntimeModelAnimation` 以副作用挂 prototype，缺即抛错 | babylon-mmd 设计 |
| WASM Bullet 独立线程，多模型共享时间轴 | MmdBulletPhysics |
| zip 条目名 Shift-JIS/UTF-8/GBK 三态，需评分择优 | 实际 `decodeZipName` |
| Vite `public/` 优先于源码同名文件 | Vite 文档 |
| Windows `os.Rename` 可能 `ERROR_SHARING_VIOLATION` | Windows 内核 |
| 模之屋 WebView 接管被 ADR-003 否决 | 项目决策 |
| `basenameFallbackFS` 解决 PMX 内路径与实际不匹配 | 实际实现 |
| Go `bool` 默认 `false`，无法区分"值为假"和"未设置" | Go 规范 |
| `OpenWithSoftware` 路径含空格被 shell 切碎，需按段替换 | 实际 bug #22 |

### 可以软（允许艺术加工）

| 可软内容 | 理由 |
|---------|------|
| 时间线——压缩/重组实际事件 | 戏剧节奏 |
| 城邦对话与性格——拟人化描写 | 增加可读性 |
| 开发者内心独白 | 叙事手段 |
| AI 的"性格"——猜错/学得快 | 已有先例 |
| 同一天事件的发现顺序 | 保持因果链即可 |
| 版本号/文件名模糊化 | 减少过时风险 |
| 中间轮次合并省略 | 避免"流水账" |

## 六、已识别的矛盾（续写时必须直面）

1. **"聚合"vs"原创焦虑"**——主角的价值是拼装，但拼装者常自我怀疑。必须让"拼得对"本身成为尊严
2. **城邦无成长弧**——必须让每个城邦在入伙前后有认知变化（从抗拒到接纳，或从乖巧到露脾气）
3. **"拼好了就行"的错觉**——同型问题会反复出现（编码、路径、副作用），第三卷必须接受这一点
4. **AI 双重性**——既是外交官又是共犯（基于记忆改、被文档骗），必须成为显性主题
5. **人类时刻被淹没**——聚合的初衷是"让用户一个入口看所有模型"，这个朴素愿望必须在技术流水账中被打捞
6. **叙事按时间而非认知阶段排列**——按认知重组会释放巨大叙事势能

## 七、续写规则

1. **先读 SKELETON.md，再读对应卷的已有章节，最后动笔**
2. **每章开头标注：第几卷、第几章、标题、对应真实事件**
3. **技术细节必须硬——参考 `docs/status.md` 的 Bug 记录与已实现清单**
4. **戏剧弧线必须软——每章要有：惊醒 → 追查 → 真相 → 教训**
5. **章尾必须有"聚合"的隐喻——指出本事件收编了哪个城邦、引入了哪条边界**
6. **禁止写"从此再无 bug"式结尾——聚合悖论是永久的**
7. **章末用 `---` 分隔，附一行教训摘要（不超过 20 字）**
8. **人类时刻优先于技术流水账——写城邦脾气之前先写"谁在受苦"**
