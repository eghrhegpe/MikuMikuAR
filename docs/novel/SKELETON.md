# 编码奇谭 · 骨架文档

> 任何 AI 续写本小说前，必须先读此文件。它钉死了世界观、冲突、主角、编年史和技术设定。

---

## 一、世界观一句话

在一个 PMX 模型散落于各城邦的世界里，一个桌面壳不发明任何东西，只让所有城邦共享同一个模型库。

它没有自己的核心能力——渲染靠 babylon-mmd，物理靠 WASM Bullet，文件靠 Go，容器靠 zip，编辑靠 Blender，资源靠模之屋。它的价值全在"搬得对、拼得稳"。它是联邦，不是帝国。

---

## 二、核心冲突：聚合悖论

每收编一个城邦，就引入它的一整套世界观与边界。联邦越强，依赖越深；城邦越独立，联邦越脆弱。聚合者无法回到"只看一个模型"的简单年代，也无法让所有城邦说同一种话。

**不可调和**：你无法不收编（聚合是它的立身之本），也无法让城邦卸下脾气（脾气就是它们的能力本身）。

| 被收编的城邦 | 带来的能力 | 带来的边界/代价 |
|------------|-----------|----------------|
| babylon-mmd | PMX 渲染 | 副作用 import，缺即崩 |
| WASM Bullet | 物理 | 独立线程、共享时间轴 |
| Go + Wails | 桌面壳 / 文件 IO | `[]byte` 经 JSON 变 base64 |
| zip 容器 | 不解压看内部 | 编码三态、zip slip |
| DanceXR 对标 | 功能蓝图 | 8 种分类硬编码 → 标签系统重构 |
| Blender | 编辑能力 | 进程启动、PATH 检测 |
| 模之屋 / 资源站 | 资源入口 | WebView 方案被否、下载监听 debounce |

## 三、主角与角色

### 主角：桌面壳的意识

MikuMikuAR 在某次 `wails dev` 扫完所有外部库后产生的自意识。

- **立场**：聚合——想成为用户打开就能看到所有模型的入口
- **欲望**：全懂——知道每个城邦的脾气、边界、文档是否过时
- **缺失**：原创——自己不发明任何东西，价值全靠拼装，常自我怀疑"我到底算什么"
- **反派**：不是某个 bug，是"文档是对的"这个信念——各城邦文档说的和实际脾气总有出入

## 五、技术设定

### 必须硬（不可虚构，否则丧失技术小说合法性）

| 硬事实 | 来源 |
|--------|------|
| `[]byte` 经 JSON 序列化返回 base64 字符串 | JSON 规范 |
| `mmdWasmRuntimeModelAnimation` 以副作用挂 prototype，缺即抛错 | babylon-mmd 设计 |
| WASM Bullet 独立线程，多模型共享时间轴 | MmdBulletPhysics |
| zip 条目名 Shift-JIS/UTF-8/GBK 三态，需评分择优 | 实际 decodeZipName |
| Vite `public/` 优先于源码同名文件 | Vite 文档 |
| Windows `os.Rename` 可能 `ERROR_SHARING_VIOLATION` | Windows 内核 |
| 模之屋 WebView 接管被 ADR-003 否决 | 项目决策 |
| `basenameFallbackFS` 解决 PMX 内路径与实际不匹配 | 实际实现 |
| Go `bool` 默认 `false`，无法区分"值为假"和"未设置" | Go 规范 |
| `OpenWithSoftware` 路径含空格被 shell 切碎，需按段替换 | 实际 bug #22 |

### 可以软（允许艺术加工）

| 可软内容 | 理由 |
|---------|------|
| 时间线——压缩/重组实际事件 | 戏剧节奏 |
| 城邦对话与性格——拟人化描写 | 增加可读性 |
| 开发者内心独白 | 叙事手段 |
| AI 的"性格"——猜错/学得快 | 已有先例 |
| 同一天事件的发现顺序 | 保持因果链即可 |
| 版本号/文件名模糊化 | 减少过时风险 |
| 中间轮次合并省略 | 避免"流水账" |

### 章的结构（每章必含）

1. **章首标题**：`# 第 N 卷 · 第 M 章 · 标题` 或 `# 第 M 章 · 标题`
2. **真实事件标注**：`> 对应真实事件：xxx`
3. **四段式叙事**：惊醒 → 追查 → 真相 → 教训
4. **章末隐喻**：一句「聚合」相关的思考
5. **教训摘要**：`---` 分隔 + 一行不超过 20 字的教训

### 技术准确性（硬约束）

- 所有技术细节必须能在 `../../` 代码库或 `../` 文档中找到依据
- 不确定的技术细节 → 查代码 → 再写，禁止脑补
- 角色性格遵循本文件角色表，不 OOC

### 叙事节奏
