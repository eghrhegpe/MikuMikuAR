# ADR-007: DanceXR 场景菜单设计参考

**日期**：2026-07-16
> **状态**: 参考文档 — 纯架构参考，无代码变更

---

### 背景

MikuMikuAR 当前阶段侧重「聚合管理器 + 播放器」，场景层面的保存/加载、舞台管理、角色与场景的操作分离尚未实现。DanceXR 作为 PMX/MMD 生态的标杆播放器，在场景 UI 和场景菜单设计上有成熟的模式可供参考。

本次决定将这些设计模式正式记录为架构参考，供后续实现场景保存/加载时使用。源文档位于 `docs/dancexr-zh/`（DanceXR 中文用户手册）。

---

### 决定

#### 1. 场景实体定义

DanceXR 中「场景」是**屏幕上所有元素的完整状态快照**：

```
场景 = 演员列表 × 每个演员的配置 × 舞台 × 动画 × 音频 × 灯光 × 摄影机 × 环境
```

保存时不包含实际模型/动作文件，仅通过内容库标识符引用。需要携带资源时用「场景包」（场景文件 + 依赖文件打包）。

**对 MikuMikuAR 的启示**：场景文件格式（如 `.mmascene`）应设计为引用库标识符而非绝对路径，确保跨设备可移植。

#### 2. 三层交互层级分离

| 层级 | 触发方式 | 包含内容 |
|------|---------|---------|
| 场景菜单 🎭 | 点击底部图标 | 舞台、道具、保存/加载场景 |
| 角色菜单 🧑 | 点击脚下选择圆盘 | 动作、材质、物理、服装、变形 |
| 角色工具菜单 🔧 | 角色菜单内螺丝刀图标 | 收藏、标签、复制、重载、移除 |

核心思想：**场景级操作 vs 演员级操作严格分家**，避免在一个菜单里堆砌所有功能。

**对 MikuMikuAR 的启示**：当前模型库弹窗 + 场景面板的扁平结构应考虑分层。场景保存/加载应作为独立菜单入口，与模型库分离。

#### 3. 底部主菜单栏 + 进度条

UI 锚定屏幕底部：
- 左侧 5 个图标：⚙️系统 → 🖼️环境 → 🎭场景 → 🎵音频动作 → 🧑角色
- 右侧控件：音量、播放列表、上/下一首
- 底部进度条：显示当前动作/音频名称 + 时间轴，点击播放/暂停，拖动快进/快退

MikuMikuAR 已有底部信息栏 + 进度条 + 播放控制，可在此基础上扩展菜单栏。

#### 4. 三态视图切换

点击空白区域循环：

| 模式 | 菜单栏 | 选择圆盘 | 用途 |
|------|--------|---------|------|
| UI 模式 | 可见 | 可见 | 正常操作 |
| 控制模式 | 隐藏 | 可见 | 拖拽/摆姿势，无菜单干扰 |
| 沉浸模式 | 隐藏 | 隐藏 | 截图/录制/纯观看 |

**对 MikuMikuAR 的启示**：Wails WebView2 环境天然支持，可作为体验打磨项。

#### 5. 选择圆盘 — 3D 空间直接操作

每个已加载角色脚下黄色圆盘：
- 点击 → 打开角色菜单
- 拖拽 → 移动
- 拖拽+滚轮 → 旋转
- 水平滚轮 → 垂直移动

这是「在 3D 空间直接操作对象」的核心交互模式，避免纯面板驱动的割裂感。

**对 MikuMikuAR 的启示**：当前通过场景面板列表切换焦点 → 自动 framing，缺少 3D 空间内的直接选择/拖拽能力。选择圆盘模式可作为 Babylon.js 的 Gizmo 或 pointer event 实现方向。
