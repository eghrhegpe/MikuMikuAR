# ADR-155: 大模型交流 — 激进路线（直接 NL 控场景）

- **状态**: 🔄 规划中（候选，非推荐）
- **日期**: 2026-07-20
- **相关**: ADR-093（声明式菜单 Schema）、ADR-154（推荐对照）、ADR-153（无障碍）

---

## 背景与问题

同 ADR-154。本路线为「跳过聊天面板过渡，直接做自然语言控场景」，面向「用自然语言直接操作 3D 场景」的体验。

## 路线对照

| 路线 | 第一步交付 | 后续扩展 | 风险 |
|------|-----------|---------|------|
| 推荐（ADR-154） | 聊天面板（客户端+流式+面板） | 叠 NL 控场景 | 🟢 低 |
| **激进（本 ADR）** | 直接 NL 控场景 | — | 🟡 中 |
| 创意（ADR-156） | 角色台词 | 接 TTS/口型 | 🟡→🔴 |

## 决策

直接实现 NL 控场景：LLM 经 function-calling 把自然语言解析为 `{action, params}` 意图，映射到 ADR-093 已有菜单动作闭集并触发对应 handler。

### 第一步交付

| 模块 | 建议落点（规划） | 内容 |
|------|----------------|------|
| 意图解析层 | `internal/app/llm/intent.go` | Go 侧 prompt + tool schema 约束，输出 `{action, params}` |
| 意图↔动作映射 | `frontend/src/core/llm-intent-map.ts` | 闭集映射，复用 ADR-093 现有 handler |
| 指令入口 UI | `frontend/src/menus/ai-command-bar.ts` | 指令输入框 / 语音入口 |

### 与推荐路线差异

省去聊天面板这一中间态，但要求「客户端 + 流式 + 意图解析 + 映射」一次性到位，首版交付面比 ADR-154 大。

## 风险与回退

| 风险 | 等级 | 缓解 |
|------|------|------|
| 模糊 / 多步指令解析失败 | 🟡 中 | tool schema 强约束 + 多步拆解为顺序意图队列 |
| 首版交付面大 | 🟡 中 | 先支持高频动作子集（灯光 / 相机 / 模型切换），余下动作灰度开放 |
| 范围同推荐 | 🟢 | 仍遵守「受限意图翻译器」原则，不开放文件 / 命令权限 |

## 实施路径

| 阶段 | 范围 | 验收 |
|------|------|------|
| Step 1 | 客户端 + 意图解析 + 映射 + 指令入口，跑通高频动作 | 自然语言触发已有菜单动作、`tsc --noEmit` 0 错、意图解析单测 |
| Step 2 | 扩展动作闭集覆盖 | 中频动作可达，误解析有兜底提示 |
