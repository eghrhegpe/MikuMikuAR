# MikuMikuAR 术语表

> UI 显示名 → 通俗解释 → 内部标识。
> 新增功能时先查此表，避免引入一般人看不懂的词。

---

## 用户可见术语

### 核心格式

| UI 显示名 | 通俗解释 | 内部标识 | 建议 |
|-----------|---------|---------|------|
| PMX 模型 | MMD 角色/舞台模型文件 | `.pmx` | 保留（核心定位词） |
| VMD 动作 | MMD 动作/相机数据文件 | `.vmd` | 解释为"动作数据" |
| MMD | MikuMikuDance，初音未来专用 3D 软件 | `mmd` | 已知生态名，保留 |
| Morph / 表情 | 模型面部表情变形 | `morph` | 统一用"表情"，代码里再用 `morph` |
| WASD | 键盘 W/A/S/D 键控制移动 | `freefly` | 可保留（游戏玩家常识） |

## 命名规范

1. **先中后英**：UI 标签优先用中文，英文技术名放括号作为补充，例如"抗锯齿 (FXAA)"
2. **不用纯英文缩写**：`PCF` → "柔和阴影"；`SSS` → "皮肤透光"
3. **代码内部标识**不受此约束，保持与 Babylon.js / MMD 生态一致（如 `morph`, `bloom`, `fxaa`）
4. **已知生态名**保留：MMD、PMX、VMD、Blender、WASD（游戏玩家/MMD 用户认知度够）
5. **数据格式不要暴露**：不在 UI 里写 `.pmx` / `.vmd` 扩展名，用"模型" / "动作"代替
