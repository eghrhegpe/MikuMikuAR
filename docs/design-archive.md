# MikuMikuAR — UI 设计归档

> 以下设计在桌面原型中探索过，最终未被采用。保留供参考。

## 1. 树 + 网格（左右分栏）

```
┌──────────────────────────────────────────────┐
│  📁 模型库                                    │
├────────────┬─────────────────────────────────┤
│  📁 角色   │  ┌──────┐ ┌──────┐ ┌──────┐   │
│    📁 Miku │  │ 🎵   │ │ 🎵   │ │ 🎵   │   │
│    📁 Rin  │  │ 模型1 │ │ 模型2 │ │ 模型3 │   │
│    📁 Len  │  └──────┘ └──────┘ └──────┘   │
│  📁 场景   │  ┌──────┐ ┌──────┐             │
│  📁 动作   │  │ 🎵   │ │ 🎵   │             │
│            │  │ 模型4 │ │ 模型5 │             │
│            │  └──────┘ └──────┘             │
├────────────┴─────────────────────────────────┤
│  共 5 个模型          Miku/                  │
└──────────────────────────────────────────────┘
```

- 左侧窄树（目录导航），右侧宽网格（内容）
- 点树节点 → 右侧只显示该文件夹内条目
- **像什么**：VS Code 侧栏、Windows 资源管理器
- **优点**：跨层跳转最快，聚焦感强
- **缺点**：面板常驻，3D 视口被挤压

## 2. 树内联嵌套卡片

```
┌──────────────────────────────┐
│  📁 模型库                    │
│                               │
│  ▼ 📁 角色                    │
│  │  ┌──────┐ ┌──────┐        │
│  │  │ 🎵   │ │ 🎵   │        │
│  │  │ Miku │ │ Rin  │ ···    │
│  │  └──────┘ └──────┘        │
│  │                           │
│  ▶ 📁 场景                   │  ← 折叠了
│                               │
│  ▼ 📁 动作                   │
│  │  ┌──────┐ ┌──────┐        │
│  │  │ 🎵   │ │ 🎵   │        │
│  │  │ 动作1│ │ 动作2│ ···    │
│  │  └──────┘ └──────┘        │
│                               │
└──────────────────────────────┘
```

- 可折叠树，文件夹节点直接展开显示卡片（缩进）
- 无独立网格区
- **像什么**：macOS Finder 列表模式、Discord 频道列表
- **优点**：层级关系一目了然
- **缺点**：嵌套三层后卡片空间被严重挤压

## 3. 面包屑 + 网格

```
┌──────────────────────────────────────────────┐
│  📁 模型库 > 📁 角色 > 📁 Miku    🔙 返回    │
├──────────────────────────────────────────────┤
│                                              │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐       │
│  │ 📁   │ │ 📁   │ │ 🎵   │ │ 🎵   │       │
│  │ 服装 │ │ 发型 │ │ 模型1│ │ 模型2│       │
│  └──────┘ └──────┘ └──────┘ └──────┘       │
│  ┌──────┐ ┌──────┐                          │
│  │ 🎵   │ │ 🎵   │                          │
│  │ 模型3│ │ 模型4│                          │
│  └──────┘ └──────┘                          │
│                                              │
└──────────────────────────────────────────────┘
```

- 顶部面包屑，下方全屏卡片网格（文件夹和模型混排）
- 双击文件夹 → 下钻，面包屑变长
- **像什么**：手机相册、iPhone 文件 App
- **优点**：卡片空间最大，最干净
- **缺点**：跨层跳转慢，需层层进退

## 选型结论

最终采用**底部抽屉式 UI**（详见 `menu-architecture.md`），以 3D 视口为主画面，UI 按需浮现，适配桌面/手机双端。
