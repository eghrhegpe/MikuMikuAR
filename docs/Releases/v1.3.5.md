# MikuMikuAR v1.3.5

**发布日期**：2026-07-13
**类型**：功能更新版

## 本版亮点

- **广场浏览器体验大幅增强（ADR-087 / ADR-099）**：`plaza.ts` 功能大幅扩展，新增广场窗口增强与相机行为扩展，配套 5 语言国际化同步更新。
- **ADR-100 相机双轴序列化双写 + 两级 UI 重构**：相机控制双轴契约落地，运行时接线（beatcut 互斥派生、订阅集中、restore 饥饿修复），UI 重构为两级结构。
- **ADR-098 babylon-mmd 对接批次一**：完成 babylon-mmd fork 的首批对接，环境模块与后端批量改动同步。
- **Go 端 COOP/COEP 注入 POC（ADR-099）**：为 `MmdWasmInstanceTypeMPR` 启用 `SharedArrayBuffer`，铺平多线程物理模拟路径。
- **GPU 资源泄漏修复**：修复 6 处 Babylon/WebAudio 对象 `dispose` 未配对泄漏。
- **缩略图渲染链路修复**：修复 live 缓存引用、异步重绘、zip 库路径透传与绝对路径回退等问题。
- **资源库健壮性**：库目录扫描与路径记忆修复，叶子目录展开一致性加固。
- **工程收敛**：ADR-101 工具逻辑收敛 wave2、ADR-102 `main.ts` 拆分、缩略图系统审计文档。

## 已知限制

| 限制 | 说明 |
|------|------|
| Windows 端目录选择器 | `SelectDir` 仍受 Wails v3 `CanChooseDirectories` 缺陷影响，弹出文件选择器而非目录选择器（待修复，见 ADR-023 §4） |
| Android localStorage 容量 | 场景自动保存写 localStorage，Android 有 5MB 限制，大场景可能写满 |
| AudioContext 惰性创建 | Android WebView 需用户交互后才能创建 AudioContext，首次无声音频可能失败 |

## 更新方式

**Windows**：下载 `MikuMikuAR-1.3.5-windows-amd64.exe`，覆盖安装即可。

**Linux**：下载 `mikumikuar-1.3.5-linux-amd64`，赋予执行权限后运行。

**Android**：下载 `MikuMikuAR-1.3.5-android-arm64.apk` 安装（Release 签名版本）。

## 升级注意

- 配置文件（`.mmascene` / 用户设置）向后兼容，无需迁移。
- 若出现异常，可尝试 `Ctrl+Shift+R` 重置本地缓存后重启。
