# ADR-116 P0+P1 实施审核报告

> 审核日期: 2026-07-17
> 审核范围: 新增 4 文件（types/registry/body-posture/hand-symmetry）+ 修改 8 文件（core/types、menu-schema、bone-override、motion-override-levels、motion-popup、model-ops、5×i18n）
> 审核方法: AGENTS.md 五步法（导入图谱 → 状态读写追踪 → 资源配对 → 心理模拟 → 报告）

## 总体结论

**有条件通过（不通过项集中在 P1 功能性缺陷）。** 架构契约、i18n 完整性、菜单路由切换、嵌套 StatePath 解析均正确；但存在 2 个 **P1 功能性缺陷**（注册表懒初始化顺序、滑块 onChange 绕过 `enabled` 门控导致状态/视觉失同步），以及 2 个 **P2**（模块 disable 误伤手动覆盖、§4.5 冲突仲裁未落地）。**P1 两项须在 P2 前修复**，否则持久化启用模块在首聚焦模型上不生效、模块开关与 3D 效果会失同步。

## 模块一：motion-modules 层（types / registry / body-posture / hand-symmetry）

**总体结论：有条件通过**

**亮点：**
- 手对称镜像数学正确：肘部 X（pitch）取反（`左ひじ[-elbowBend]`→`右ひじ[elbowBend]` 取反）、腕部 Y（yaw）保持+`mirrorOffset`、腕部 Z（roll）取反 → 与 ADR §三「镜像规则精确约定」一致（`hand-symmetry.ts:33-38`）。
- 身体姿态骨骼映射与 ADR §三示例一致（`上半身`←tilt、`腰`←bend、`上半身2`←twist）（`body-posture.ts:24-33`）。
- 模块实例设计为**无状态转换器**，状态全存 `ModelInstance.motionOverrideModules`，符合 ADR「状态来源唯一」原则。
- `bake()` 复用引擎 `_overrideMap`（O(1)），无额外每帧开销。

**风险：**

| 文件 | 观察 | 建议 |
|------|------|------|
| 🔴 P1 | `registry.ts` + `model-ops.ts:61` | `initMotionModules()` 仅在 `motion-override-levels.ts:38`（面板首次打开）调用；`focusModel` 在更早的启动/聚焦时调用 `setTargetModel` 面对**空注册表** → 已持久化 `enabled` 的模块在首聚焦模型上**永不烘焙**（开关显示 ON 但 3D 无效果，除非用户切换模型或手动开关）。 | 在 app 启动/bootstrap 调用 `initMotionModules()`；并作为防御在 `setTargetModel` / `createModule` 入口首行调用 `initMotionModules()`（幂等）。 |

## P1 推荐修复 diff（必须项）

### 修复 A：注册表初始化时机（registry.ts）

```ts
// registry.ts —— 在 setTargetModel / createModule 入口防御性确保注册
export function setTargetModel(modelId: string | null): void {
    initMotionModules();              // ← 新增：幂等，确保注册表已就绪
    if (_currentModelId === modelId) { return; }
    // ...其余不变
}

export function createModule(id: string, modelId: string): MotionOverrideModule | null {
    initMotionModules();              // ← 新增
    const entry = _registry.get(id);
    if (!entry) { return null; }
    return entry.factory(modelId);
}
```

> 同时建议在 app bootstrap（主入口或 model-manager 初始化处）显式调用一次 `initMotionModules()` 作为主初始化路径，上面的防御调用为兜底。

### 修复 B：滑块 onChange 走 setParam + 启用门控

`body-posture.ts` / `hand-symmetry.ts` 每个滑块 `onChange` 改为：

```ts
onChange: (v) => {
    createBodyPostureModule(modelId).setParam('tilt', v as number);
    // 旧代码 setModuleParam(modelId, MODULE_ID, 'tilt', v as number); bake(modelId); 删除
},
```

`setParam` 改为门控 + 自动启用（两模块相同模式）：

```ts
setParam(name: string, value: ParamValue): void {
    setModuleParam(modelId, MODULE_ID, name, value);
    const st = getModuleState(modelId, MODULE_ID);
    if (!st.enabled) { st.enabled = true; }   // 拖滑块即视为启用意图（VRChat 行为）
    bake(modelId);
}
```

> 配套：`disable()` 在 `enabled=false` 后清除 `ownedBones`（见 P2 修复）；
> `enable()` 保持 `enabled=true` + `bake`。如此开关/滑块/3D 三者一致，无孤儿覆盖。
