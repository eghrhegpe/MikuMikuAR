一、scene.ts — 渲染状态层（核心）
1.1 创建 DefaultRenderingPipeline（scene.ts:49 后新增）

ts
import "@babylonjs/core/PostProcesses/RenderPipeline/Pipelines/defaultRenderingPipeline";
export const pipeline = new DefaultRenderingPipeline("default", true, scene, [scene.activeCamera]);

相机陷阱处理：switchCameraMode（camera.ts:117-181）每次切相机会 removeCamera+新建。在 camera.ts 末尾或 scene.ts 暴露 reattachPipeline()，在 switchCameraMode 末尾调用：pipeline.cameras = [scene.activeCamera]。需让 camera.ts 能拿到 pipeline——方案：scene.ts export pipeline，camera.ts import 它（目前 camera.ts 不 import scene.ts，但 import 一个 const 无副作用，可行）。
1.2 RenderState 接口 + get/set（scene.ts，仿 LightState）

ts
export interface RenderState {
    // 后处理
    bloomEnabled: boolean;
    bloomWeight: number;      // 0-1, 默认 0.3
    bloomThreshold: number;   // 0-1, 默认 0.5
    bloomKernel: number;      // 0-512, 默认 64
    outlineEnabled: boolean;
    outlineColor: [number, number, number];  // RGB 0-1
    fxaaEnabled: boolean;
    colorCorrectionEnabled: boolean;
    // 舞台 / imageProcessing
    toneMapping: number;      // 0=OFF 1=ACES 2=Reinhard 3=Cineon 4=Neutral (对应 Babylon 枚举)
    exposure: number;         // 0-4, 默认 1
    contrast: number;         // 0-4, 默认 1
    fov: number;              // 0.1-3 rad, 默认 0.8
    bgColor: [number, number, number];  // 0-1
}
getRenderState(): RenderState — 读 pipeline + scene + activeCamera 当前值
setRenderState(s: Partial<RenderState>): void — 应用到 pipeline / scene.imageProcessingConfiguration / activeCamera.fov
toneMapping 映射：scene.imageProcessingConfiguration.toneMappingType = ImageProcessingConfiguration[toneMapping枚举]
FOV：scene.activeCamera.fov = s.fov（需判 activeCamera 非空；switchCameraMode 后新相机也要应用——见 1.3）

不需要改模型加载（材质不在本次范围）。
1.4 轮廓线（Outline）
Babylon 的 DefaultRenderingPipeline 没有原生 Outline 后处理。两种方案：

方案 A（推荐）：用 mesh.enableEdgesRendering() + mesh.edgesWidth + scene.clearColor 对比。但这是逐 mesh 的，不是后处理。
方案 B：用 babylon-mmd 的卡通描边（MmdStandardMaterial 有 outline 属性）。
本次决定：用 法线/深度边缘检测后处理（Babylon 社区有 OutlinePostProcess 但非官方）。为控制范围，本次「轮廓线」降级为 outlineEnabled 时对所有 mesh 开 edgesRendering（简单可靠），真正的后处理描边留待材质那轮一起做。UI 上标注「边缘高亮」。
1.5 色彩校正
DefaultRenderingPipeline.imageProcessing 支持 contrast，已纳入 RenderState。完整色彩校正（色相/饱和度）需额外 ColorCorrectionPostProcess——本次只暴露 contrast + toneMapping（覆盖 90% 需求），高级调色留后期。

风险点
相机重挂：switchCameraMode 后 pipeline.cameras 未更新 = 后处理失效。已在 1.3 处理，camera.ts 末尾加一行重挂。
轮廓线方案降级：本次用 edgesRendering 代替真后处理描边，效果不如真正的 outline postprocess，但零依赖、稳定。UI 文案标注「边缘高亮」避免误导。
MmdStandardMaterial 与边缘：edgesRendering 对 MMD mesh 可能需逐 mesh 开启，outlineEnabled 切换时遍历 modelRegistry 所有 inst.meshes。
预设参数范围：用户保存预设时只快照后处理+舞台参数（不含材质，符合本次范围）。预设 JSON 结构留 params map 方便后期加材质。
FOV 在 ArcRotateCamera：Arc 相机的 fov 主要影响透视感，与 FreeCamera 的 fov 语义略不同——滑块统一设 activeCamera.fov，两种相机都生效但视觉差异需手测接受。
