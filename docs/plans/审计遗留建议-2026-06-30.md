# 审计遗留建议清单

> 整理自 2026-06-30 全模块审计。已修复 80+ 项，以下为**延后/未实施**的建议，按优先级和模块分组。

---

## 🔴 高风险（建议排入下一轮）

| 模块 | 问题 | 说明 |
|------|------|------|
| `settings.ts` | XSS：`innerHTML` 拼接未全面转义 | 数十处 `innerHTML` 字符串拼接，涉及 `buildSettingsSoftwareLevel`/`buildSoftwareDetailLevel`/`addToggleRow` 等。需全面改用 `createElement`+`textContent` 或 `escapeHtml`。**建议排入专项重构** |
| `model-preset.ts` | `applyPresetFromLib` 中 `loadPMXFile` 后用路径查模型 | 路径归一化可能不匹配。建议 `loadPMXFile` 返回模型 ID 直接使用 |
| `model-preset.ts` | 材质启用状态（`isMatEnabled`）未序列化 | `ModelPresetFile` 缺少 `materialEnabled` 字段，预设恢复时材质可见性丢失 |

---

## 🟡 中等优先级

### 性能 & 体验

| 模块 | 问题 |
|------|------|
| `scene-loader.ts` | 缩略图捕获用双 rAF，低端 GPU 可能过早。建议用 `scene.whenReadyAsync()` |
| `xpbd-cloth.ts` | `_updateClothMesh` 每帧分配 `new Float32Array`（288 粒子 ~3.4KB），建议缓存复用 |
| `xpbd-collider.ts` | `updateMatrices` 每帧归一化方向向量（骨骼 Y 轴通常已是单位向量） |
| `library-core.ts` | 大量模型列表构建时同步 DOM 操作可能卡顿，建议虚拟滚动或 rAF 分片 |
| `motion-popup.ts` | `buildDanceSetsOverviewLevel` async render 无加载占位，后端慢时空白闪烁 |
| `scene-menu.ts` | `buildPresetScenesLevel` async renderCustom 同样问题 |

### 状态一致性

| 模块 | 问题 |
|------|------|
| `cloth-manager.ts` | `recreateCloth` 在 `clothEnabled=false` 时静默返回，调用方误解 |
| `scene-env-bridge.ts` | `_timeOfDayTick` 每帧调 `redoEnvAutoLink`（虽内部有 guard），可加阈值 |
| `scene-env-bridge.ts` | `_envPersistTimer` 防抖策略——页面关闭前定时器未触发则丢失状态。建议加 `beforeunload` |
| `library-core.ts` | `refreshLibrary` 强制回到顶层丢失导航深度（设计决策，但用户体验差） |
| `model-detail.ts` | 变换面板滑条不会随快捷键/外部修改自动刷新 |
| `scene-lighting.ts` | 阴影在 `dirIntensity<0.1` 时自动关闭，动画过渡中突然消失不自然 |

### 崩溃防护

| 模块 | 问题 |
|------|------|
| `model-detail.ts` | `buildModelInfoLevel` 中 `inst.mmdModel?.runtimeBones?.length` 对 Stage 模型用可选链（✅已修一处，复查其余） |

## 🟢 低优先级（代码质量/微优化）

### 类型与安全

| 模块 | 问题 |
|------|------|
| `scene-lighting.ts` | `_envSysShadow` 类型从 `any` 改为 `ShadowGenerator \| null`（✅已修） |
| `scene-lighting.ts` | `getLightState` 在未初始化时返回硬编码默认值，与模块变量可能不一致 |
| `xpbd-collider.ts` | 碰撞刚度 `stiffness` 未考虑粒子质量的加权 |
| `motion-popup.ts` | `motionMenu` 空引用风险——回调中先判空 `if (menu) menu.reRender()` |
| `camera.ts` | Concert 模式用 `0.016` 硬编码帧时间，建议用 `scene.deltaTime / 1000` |
| `camera.ts` | `modelRegistry` 依赖全局 live binding，与已迁移模块不一致（建议统一注入 ModelManager） |
| `scene-env-impl.ts` | 大量未使用导入（`Vector2`, `ParticleSystem`, `DirectionalLight` 等） |
| `model-material.ts` | 未使用导入 `Material`、`StandardMaterial` |
| `library-core.ts` | 未使用导入 `escapeHtml`、`thumbnailCache`、`resetModelMorphs` 等 |
| `outfit.ts` | `_getSlotFor` 中 `byCategory`/`all` 字段用 `as any`，建议完善 `OutfitVariant` 接口 |

### 架构与重构

| 模块 | 问题 |
|------|------|
| `scene/` 目录 | 22 个文件平铺，建议按 domain 分子目录（`model/`/`env/`/`lighting/`/`vmd/`） |
| `xpbd-collider.ts` | `updateCapsuleSizes` 功能已验证可用（骨骼名修复），但未被 `cloth-manager` 实际调用 |
| `scene-env-water.ts` | 焦散纹理 `_causticTexture` 在 `disposeWater` 中释放，切换粒子类型时重复生成 |
| `env-menu.ts` | `buildPropDetailLevel` 中 5 个滑块可提取为配置数组循环 |
| `motion-popup.ts` | `createNewDanceSet` 中 `DanceSet` ID 使用简单哈希，可能存在碰撞 |
| `model-detail.ts` | `buildModelInfoLevel` 中内联样式过多，可抽取为 CSS 类 |
| `model-preset.ts` | `buildPresetListLevel` 内联样式过多，事件监听未用委托模式 |
| `settings.ts` | `addCsRow` 自定义滑动条无拖拽功能，建议改用原生 range input |
| `settings.ts` | 性能模式切换后未持久化到后端配置（需新增 Go binding） |
| `settings.ts` | 下载监听的 `dirInput` 引用在卡片重建后可能失效 |

### UI 细节

| 模块 | 问题 |
|------|------|
| `menu.ts` (SlideMenu) | 键盘焦点在鼠标悬停后丢失，建议恢复机制 |
| `menu.ts` (SlideMenu) | 移动端 `mouseenter`/`mouseleave` 可能闪烁提示 |
| `menu.ts` (SlideMenu) | 过渡时间硬编码为 `0.12s`/`0.15s`，建议统一为 CSS 常量 |
| `outfit-ui.ts` | 点击变体后无 loading 视觉反馈 |
| `scene-env-water.ts` | `addRipple` 涟漪重用逻辑在极端多涟漪下可能误覆盖 |
| `procedural-motion.ts` | `MORPH_BLINK_CANDIDATES` 不匹配时静默无眨眼，建议加 `console.debug` |

### 功能扩展

| 模块 | 功能 |
|------|------|
| `xpbd-cloth.ts` | 仅支持 `skirt` 拓扑，接口预留了 `tube`/`cape`/`rope` |
| `motion/lipsync.ts` | 仅控制「あ」口型，不支持圆唇音「お」多 morph 映射 |
| `motion/vmd-writer.ts` | 关键帧插值参数硬编码为线性，可扩展支持贝塞尔缓入缓出 |
| `model-preset.ts` | 预设中未包含布料配置/物理开关 |
| `model-preset.ts` | `_presetUndoStack` 仅支持单模型撤销 |
