# 🧪 修复周期与验收契约

修复一个 Bug 或未达标的实现时，走以下 5 步。**第 3 步（验收契约）是核心** — AI 在写代码前押注「什么证据证明修好了」，避免先写后猜。

---

## 流程

```
你：描述问题
   ↓
AI：写复现步骤 + 验收契约
   ↓
你：确认契约（或补充修正）
   ↓
AI：实现修复
   ↓
你：按契约验证 → 通过 / 驳回
```

---

## 模板

每次修复会话，AI 先按此模板输出，经确认后再写代码。

### 1. 问题描述（你）

<运行环境、错误现象、预期行为>

### 2. 复现步骤（AI 写，你确认）

```
1. 具体操作路径
2. 观察点（控制台输出 / UI 状态 / 文件变化）
3. 失败表现
```

### 3. 验收契约（AI 写，你确认）

> 🔴 **修复是否完成，以此处定义的证据为准。** 构建通过不算完成。

✅ 证据 1：<具体的可观察输出，如 `控制台出现 "FS: fallback …"`>
✅ 证据 2：<第二个证据，如有>
❌ 不纳入验收：<排除项，如「纹理完美无损」等当前不要求的>

### 4. 修复实现

[代码变更]

### 5. 验证结果（你）

```
通过 / 驳回：
[贴证据]
```

---

## 快速示例

### 1. 问题描述
加载「诗蔻蒂」模型，模型白色无纹理。控制台无 `FS:` 相关日志。

### 2. 复现步骤
```
1. wails dev 启动
2. 点击「打开 PMX」，选择 model.pmx
3. 打开 DevTools Console
4. 过滤 FS: 关键词
```

### 3. 验收契约
✅ 控制台出现 `"FS: fallback /textures/face.bmp → TEX1.png"` 日志
✅ 模型网格的 `diffuseTexture` 有值（诊断日志输出）
❌ 不要求 TGA 格式纹理

---

## 何时用

- **修复 Bug** — 必须走完整流程
- **实现新功能** — 可以在计划阶段引用契约，但不需要逐条输出
- **「试试看」的探索** — 不走，先写代码再讨论
