# MikuMikuAR Design Context

## Users
MMD 爱好者自用高精度查看器，未来做可分发产品面向同好/创作者。UI 要够克制不抢戏（自用），但质感要经得起截图/录屏传播（产品化）。

## Brand Personality
**沉浸 · 冷静 · 有温度**

- 沉浸：全屏 Babylon 渲染不被 UI 打断，浮层尽可能透明、延迟出现
- 冷静：暗色、低饱和蓝紫、无刺眼高亮
- 有温度：微妙的 `rgba(74,108,247,…)` 强调色，不是冰冷黑白

## Aesthetic Direction
- 参考：DanceXR（分类清晰、克制）、Blender Shading Workspace（暗色、工具随需出现）、VSCode 深色主题（Design Token 思路）
- 反参考：MMD 原版主界面（过彩、过碎）、国产播放器花哨皮肤、浅色 Office Ribbon 风格
- 主题：仅暗色

## Design Principles
1. **Canvas first** — 3D 画面是主角，UI 是配角。浮层半透明，控件按时出现、及时消失。
2. **One action, one surface** — 每个操作仅展开它需要的浮层，不堆叠无关信息。
3. **Subtle accent** — 蓝色强调 `#4a6cf7` 克制使用，不喧宾夺主。
4. **Desktop-first** — Wails 桌面应用优先，键盘快捷键一等公民，不为手机妥协。
5. **Screenshot-ready** — 任何状态截图都应该是干净的，没有未完成感。
